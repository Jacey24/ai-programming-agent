# 内部控制台使用指南 — AI Agent 快速定位 API 方法

## 一句话说明

Server 进程内置了完整 CLI 测试环境（`--console` 参数），所有对外 API 的调用路径集中在 3 个文件中。

---

## 如何让 AI Agent 快速找到"所有 API 在哪调用"

### 1. 所有 HTTP 路由 → 一行命令定位

```bash
# 让 Agent 读取这个文件，它是所有路由的唯一注册点
cat backend/agent-server/api/HttpServer.cpp
```

路由格式统一：
```cpp
svr.Get(R"(/api/v1/tasks/([a-zA-Z0-9\-_]+))", ...)  → TaskController::getTask()
svr.Post("/api/v1/tasks", ...)                        → TaskController::createTask()
svr.Get("/api/v1/experts", ...)                       → ExpertController::listExperts()
```

### 2. 所有 Controller 头文件 → API 能力声明

```bash
ls backend/agent-server/api/controllers/*.h
```

| 头文件 | 对外 API 模块 |
|--------|-------------|
| `TaskController.h` | 任务 CRUD + cancel + active |
| `ExpertController.h` | Expert CRUD + graph + tools/routes/llm 子资源 |
| `ConfigController.h` | 7 种配置文件 GET/PUT + LLM connection test + provider CRUD |
| `SessionController.h` | 对话 CRUD |
| `GlobalController.h` | 全局上下文 CRUD |
| `WorkspaceController.h` + `WorkspaceFileController.h` | 工作区 + 文件树 |
| `PermissionController.h` | 权限审批 |
| `DebugController.h` | 调试器断点/单步 |
| `ToolController.h` | 工具列表查询 |

### 3. 内部控制台（CLI）→ API 实际调用范例

```bash
# 控制台是"所有 API 的完整调用演示"
cat apps/agent-server/InternalConsole.cpp
```

每个 `handle*()` 方法展示了一个 API 的完整调用链：
```
/global list    → doGet("/api/v1/globals")         → parseOrError()
/task create    → doPost("/api/v1/tasks", body)    → startStreamListener()
/workspace list → doGet("/api/v1/workspaces")      → 打印 items
```

## 启动方式

```powershell
.\build_win\apps\agent-server\Release\codepilot-agent-server.exe --console
```

启动后会在同终端出现 `> ` 提示符，可直接输入命令测试所有 API。

## 可用命令一览

| 命令 | 测试的 API |
|------|-----------|
| `/global list` | `GET /api/v1/globals` |
| `/global create <name>` | `POST /api/v1/globals` |
| `/global use <id>` | `GET /api/v1/globals/:id` |
| `/global delete <id>` | `DELETE /api/v1/globals/:id` |
| `/workspace create <name> <path>` | `POST /api/v1/workspaces` |
| `/workspace list` | `GET /api/v1/workspaces` |
| `/workspace files [id]` | `GET /api/v1/workspaces/:id/files/tree` |
| `/task create <goal>` | `POST /api/v1/tasks` + SSE 监听 |
| `/task list` | `GET /api/v1/tasks` |
| `/task status <id>` | `GET /api/v1/tasks/:id` |
| `/task cancel <id>` | `POST /api/v1/tasks/:id/cancel` |
| `/task history <id>` | `GET /api/v1/tasks/:id/events/history` |
| `/session create <title>` | `POST /api/v1/sessions` |
| `/session list` | `GET /api/v1/sessions` |
| `/perm list` | `GET /api/v1/permissions` |
| `/perm approve [id]` | `POST /api/v1/permissions/:id/approve` |
| `/tools` | `GET /api/v1/tools` |
| `/experts list` | `GET /api/v1/experts` |
| `/health` | `GET /health` |
| `/verbose on` | 开启 HTTP 请求日志 |

## AI Agent 使用技巧

给 AI Agent 的 prompt 模板：

> "请阅读 `backend/agent-server/api/HttpServer.cpp` 获取所有路由注册点，然后阅读对应的 `apps/agent-server/InternalConsole.cpp` 获取每个 API 的实际调用范例，最后在对应的 Controller 头文件中确认参数签名。所有 API 参照见 `docs/全API参照表.md`。"

这个 3 文件定位法覆盖了项目的所有对外 API 接口。

## 三、Pause/Resume 机制（核心新增）

### 3.1 动机
当前仅支持 cancel（单向销毁），缺少 pause 状态。Pause 需同时服务于：
- **用户主动暂停**：阻断循环、缓存上下文、允许追加消息
- **权限申请**：替代 `PermissionManager::waitForResolution()` 的同步阻塞

### 3.2 TaskSnapshot（已实现）

定义于 `domain/agent/AgentLoopResumer.h`，包含 runExpertChain() 所有循环变量和上下文，
可序列化到 DB 或 JSON。

### 3.3 ResumeUtil：单函数黑盒（已实现）

**文件**：`domain/agent/AgentLoopResumer.h/.cpp`

**组员 A 唯一的调用入口**：
```cpp
auto result = ResumeUtil::prepareResume(snapshot, userMessage);
// 内部自动判断：无用户消息 → 直接恢复 sessionHistory
//             有用户消息 → 自动构建 _resumer Expert → 调 LLM → 更新 snapshot
```

**返回值** `ResumeResult`：
- `snapshot` — 修改后的快照（sessionHistory/summary/currentExpert 已更新）
- `handled` — 是否已完成（_resumer 直接完成时无需继续执行）
- `errorMessage` — 错误信息

**内部逻辑**：
1. `userMessage` 为空 → 直接返回原 snapshot，无需 _resumer
2. `userMessage` 非空 → 构建 _resumer Expert（硬编码，不依赖 config）→ 调 LLM
3. 解析 _resumer 输出的 `<write to="summary">` 和 `<done>/<fail>` 标签
4. 更新 sessionHistory → 通过 RouteEngine 路由到下一个 Expert → 返回修改后的 snapshot

组员 A 无需知道 _resumer 的存在，也无需编写任何路由逻辑。

### 3.4 AgentLoop 配合改动（由组员 A 实施）

1. `runExpertChain()` 新增参数 `const std::string& initialSessionHistory = ""` — 已修改 `AgentLoop.h`
2. Pause 检查点：保存 `TaskSnapshot` → 返回 `status="paused"`
3. Resume 入口：调用 `ResumeUtil::prepareResume()` → 解包 snapshot → 重新进入 `runExpertChain()`

### 3.5 与权限的统一

```
旧：callToolWithPermission() → waitForResolution() 同步阻塞
新：callToolWithPermission() → 触发 pause → 前端展示 + 用户追加消息
    → ResumeUtil::prepareResume(snapshot, message) → runExpertChain()
```

## 四、权限系统

### 4.1 批处理（基于 Pause）
Pause 机制就绪后，LLM 输出的 N 个 `<cmd>` 可一次性打包为 PermissionBatch，前端统一审批。

### 4.2 分层权限策略

```
全局默认策略（config/tools.json）
    ↓ 继承
Workspace 级别覆盖（workspaces.permission_overrides JSON 字段）
    ↓ 合并（取最严格值）
Agent 执行时 callToolWithPermission()
```

- 数据库改动：`ALTER TABLE workspaces ADD COLUMN permission_overrides TEXT DEFAULT '{}';`
- 新增端点：`GET/PUT /api/v1/workspaces/:id/permissions`

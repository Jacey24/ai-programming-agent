# CodePilot 环境配置说明

本文档用于统一小组成员的本地开发环境。组员 Fork 或 Clone 项目后，应先按本文档完成环境安装、项目启动和联调验证，再开始开发。

当前项目采用 Docker 统一运行环境：

```text
前端：Nginx 托管静态 Web 页面
后端：C++20 Agent Server
数据库：SQLite
构建：CMake + Ninja
部署：Docker Desktop + Docker Compose
```

---

## 1. 项目技术栈

### 1.1 前端

- HTML
- CSS
- JavaScript
- Nginx 静态文件服务

前端目录：

```text
frontend/web/
├── index.html
├── styles.css
└── main.js
```

当前前端功能：

- 查看后端服务状态
- 查看数据库连接状态
- 输入用户提示词
- 显示 AI 返回结果
- 查看历史聊天记录
- 查看接口响应 JSON

当前 AI 业务逻辑尚未实现，用户输入任意内容后，后端固定返回：

```text
待开发
```

### 1.2 后端

- C++20
- CMake
- Ninja
- SQLite C API
- 原生 Socket HTTP Server

后端入口目录：

```text
apps/agent-server/
├── main.cpp
├── AppBootstrap.cpp
└── CMakeLists.txt
```

当前后端已实现接口：

```text
GET  /api/v1/health
POST /api/v1/chat
GET  /api/v1/chat/history
```

### 1.3 数据库

- SQLite
- 数据库文件：`storage/agent.db`
- 容器内路径：`/data/agent.db`

当前主要表：

```text
system_health
sessions
tasks
chat_messages
```

聊天历史表：

```sql
CREATE TABLE IF NOT EXISTS chat_messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    prompt TEXT NOT NULL,
    response TEXT NOT NULL,
    created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 启动前必须确认

启动项目前，请确认：

1. Docker Desktop 已安装。
2. Docker Desktop 已打开。
3. Docker Engine 正在运行。
4. 当前命令行位于项目根目录。
5. 项目中存在 `deploy/docker/docker-compose.yml`。

检查 Docker：

```bash
docker --version
docker compose version
```

如果命令无法识别，说明 Docker Desktop 没有正确安装，或命令行需要重新打开。

---

## 5. 一键启动前端和后端

在项目根目录执行：

```bash
docker compose -f deploy/docker/docker-compose.yml up --build -d codepilot-server codepilot-web
```

该命令会启动：

```text
codepilot-server  后端服务，端口 8080
codepilot-web     前端服务，端口 8081
```

启动成功后访问：

```text
前端页面：http://127.0.0.1:8081/
后端接口：http://127.0.0.1:8080/api/v1/health
```

## 6. 实时查看后端日志

```bash
docker compose -f deploy/docker/docker-compose.yml up --build codepilot-server
```

```text
Ctrl + C
```

```bash
docker compose -f deploy/docker/docker-compose.yml logs -f codepilot-server
```

或者：

```bash
docker logs -f codepilot-server
```

## 8. 验证后端服务

Windows PowerShell：

```powershell
curl.exe http://127.0.0.1:8080/api/v1/health
```

macOS / Linux：

```bash
curl http://127.0.0.1:8080/api/v1/health
```

正常返回示例：

```json
{
  "success": true,
  "data": {
    "status": "ok",
    "service": "codepilot-agent-server",
    "version": "0.1.0",
    "database": {
      "type": "sqlite",
      "connected": true,
      "path": "/data/agent.db"
    }
  }
}
```

如果 `database.connected` 为 `true`，说明后端已经成功连接 SQLite。

## 11. 验证数据库

查看数据库文件：

```bash
docker exec codepilot-server ls -l /data
```

查看数据表：

```bash
docker exec codepilot-server sqlite3 /data/agent.db ".tables"
```

正常应看到：

```text
chat_messages  sessions  system_health  tasks
```

查看聊天历史数据：

```bash
docker exec codepilot-server sqlite3 /data/agent.db "select id, prompt, response, created_at from chat_messages order by id desc;"
```

如果能看到刚才前端或接口提交的记录，说明历史记录已经成功写入数据库。

## 12. 停止项目

停止前端和后端：

```bash
docker compose -f deploy/docker/docker-compose.yml down
```

停止后，SQLite 数据库文件仍保留在：

```text
storage/agent.db
```

下次启动仍可看到历史聊天记录。

---

## 13. 清空数据库并重新初始化

如果需要清空本地数据库，先停止容器：

```bash
docker compose -f deploy/docker/docker-compose.yml down
```

删除数据库文件：

Windows PowerShell：

```powershell
Remove-Item .\storage\agent.db
```

macOS / Linux：

```bash
rm ./storage/agent.db
```

重新启动：

```bash
docker compose -f deploy/docker/docker-compose.yml up --build -d codepilot-server codepilot-web
```

后端启动时会自动重新创建 SQLite 数据库和基础表。

## 15. 当前接口约定

### 15.1 健康检查

```http
GET /api/v1/health
```

用途：

- 检查后端是否启动
- 检查 SQLite 是否连接成功

### 15.2 创建聊天记录

```http
POST /api/v1/chat
Content-Type: application/json
```

请求体：

```json
{
  "prompt": "用户输入内容"
}
```

响应：

```json
{
  "success": true,
  "data": {
    "id": 1,
    "prompt": "用户输入内容",
    "response": "待开发"
  }
}
```

说明：

- 当前不调用真实 AI。
- 后端固定返回 `待开发`。
- 每次调用都会写入 `chat_messages` 表。

### 15.3 查询历史聊天记录

```http
GET /api/v1/chat/history
```

响应：

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": 1,
        "prompt": "用户输入内容",
        "response": "待开发",
        "created_at": "2026-07-06 11:09:04"
      }
    ]
  }
}
```

## 16. 组员开发流程

每次开始开发前：

```bash
git pull
```

从主分支创建自己的功能分支：

```bash
git checkout -b feat/功能名称
```

示例：

```bash
git checkout -b feat/chat-history-page
git checkout -b feat/api-session
git checkout -b feat/tool-system
```

开发完成后：

```bash
git status
git add .
git commit -m "feat: 完成历史记录页面"
git push
```

然后在 GitHub 上提交 Pull Request。

## 18. 常见问题

### 18.1 Docker 命令无法执行

检查 Docker Desktop 是否已经启动。

```bash
docker --version
docker compose version
```

如果仍然失败，关闭并重新打开 PowerShell 或终端。

### 18.2 端口被占用

后端端口：

```text
8080
```

前端端口：

```text
8081
```

如果启动失败，检查是否有其他程序占用端口。

Windows PowerShell：

```powershell
netstat -ano | findstr :8080
netstat -ano | findstr :8081
```

### 18.3 前端页面打不开

检查前端容器：

```bash
docker ps
docker logs codepilot-web
```

确认浏览器访问：

```text
http://127.0.0.1:8081/
```

### 18.4 后端接口访问失败

检查后端容器：

```bash
docker ps
docker logs codepilot-server
```

验证健康检查：

```bash
curl http://127.0.0.1:8080/api/v1/health
```

### 18.5 数据库没有历史记录

先在前端发送一次提示词，或者调用：

```powershell
$body = @{ prompt = "测试历史记录" } | ConvertTo-Json -Compress
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:8080/api/v1/chat -ContentType "application/json" -Body $body
```

然后查询：

```bash
docker exec codepilot-server sqlite3 /data/agent.db "select * from chat_messages;"
```

### 18.6 修改前端后浏览器没变化

前端文件通过 volume 挂载到 Nginx 容器，一般刷新浏览器即可。

如果仍然没变化：

```bash
docker compose -f deploy/docker/docker-compose.yml restart codepilot-web
```

### 18.7 修改后端代码后没生效

后端需要重新构建镜像：

```bash
docker compose -f deploy/docker/docker-compose.yml up --build -d --force-recreate codepilot-server
```

## 19. 最小联调链路

组员本地至少要跑通以下链路：

1. Docker Desktop 启动成功。
2. `codepilot-server` 容器启动成功并显示 healthy。
3. `codepilot-web` 容器启动成功。
4. 浏览器能打开 `http://127.0.0.1:8081/`。
5. 前端显示后端服务运行中。
6. 前端显示数据库已连接。
7. 输入提示词并发送，AI 返回 `待开发`。
8. 历史记录区域能看到刚才的对话。
9. SQLite 表 `chat_messages` 中能查到该记录。

如果以上 9 步全部通过，说明本地开发环境配置成功。

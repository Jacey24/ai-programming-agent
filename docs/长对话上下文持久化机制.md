# 长对话上下文持久化机制（v2 — Global 架构重构）

> **重大变更**：v2 将原 `Session` 概念升级为 `Global`（项目/主题级），去除了 `parentTaskId` 任务串联机制，改为 `global_context` 知识库检索。

---

## 1. 三层上下文定义

```
Global (g_xxx) — 项目/主题级（持久化）
  │  • globals 表 + global_context 表
  │  • 跨 Task 的知识库：summary、plan、output 归档
  │  • 手动创建 API，也可程序自动创建
  │  • SSE 推送 global 级事件
  │
  ├── Task A (t_xxx) — 单次调用，独立生命周期
  │     ├── Expert Chain 执行
  │     ├── sessionHistory（内部自我上下文，不暴露、不持久化）
  │     └── 结束时 summary → global_context
  │
  ├── Task B (t_xxx)
  │     ├── <read from="global"> 检索历史知识
  │     └── 无 parentTaskId 串联
  │
  └── Task C ...
```

| 层次 | 作用域 | 持久化 | 暴露给外部 |
|------|--------|--------|-----------|
| **sessionHistory** | Expert 内部 | ❌ 不持久化 | ❌ 不暴露 |
| **Summary** | Task 内跨 Expert | ✅ 写入 global_context | ✅ 通过 global_context API |
| **Global** | 跨 Task 项目级 | ✅ globals + global_context 表 | ✅ 完整 CRUD API |

---

## 2. ID 体系

| ID | 前缀 | 示例 | 说明 |
|----|------|------|------|
| global_id | `g_` | `g_default` | Global（项目/主题）主键 |
| task_id | `t_` | `t_a1b2c3d4` | Task 主键 |
| workspace_id | `w_` | `w_e5f6g7h8` | Workspace 主键（保留兼容） |

## 3. 数据库 Schema

### 3.1 globals 表（替代旧 sessions 表）

```sql
CREATE TABLE IF NOT EXISTS globals (
    id TEXT PRIMARY KEY,              -- g_xxx（默认启动时自动创建 g_default）
    name TEXT NOT NULL,               -- 项目/主题名称
    description TEXT DEFAULT '',      -- 描述
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

### 3.2 global_context 表（新增 — Global 知识库）

```sql
CREATE TABLE IF NOT EXISTS global_context (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    global_id TEXT NOT NULL,          -- 关联 globals
    source_task_id TEXT NOT NULL,     -- 来源 Task
    type TEXT NOT NULL,               -- 'summary' | 'plan' | 'output'
    content TEXT NOT NULL,            -- 内容
    created_at TEXT NOT NULL,
    FOREIGN KEY (global_id) REFERENCES globals(id)
);
```

### 3.3 tasks 表（字段改名）

```sql
-- session_id → global_id
CREATE TABLE IF NOT EXISTS tasks (
    id TEXT PRIMARY KEY,
    global_id TEXT NOT NULL,          -- ★ 替代原 session_id
    workspace_id TEXT NOT NULL,
    goal TEXT NOT NULL,
    status TEXT NOT NULL,
    plan TEXT,
    current_step TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

## 4. 核心逻辑

### 4.1 默认 Global

系统启动时自动检查 `globals` 表：
- 若为空 → 自动创建 `g_default`（名称为 "默认工作区"）
- 前端/TUI 不传 `global_id` 时，默认使用 `g_default`

### 4.2 Task 创建

```
POST /api/v1/tasks
{
  "global_id": "g_default",     // ★ 新字段（兼容旧字段 session_id）
  "workspace_id": "w_xxx",
  "input": "帮我写一个排序算法"
}
```

**兼容性**：接受 `session_id` 作为 `global_id` 的别名。若两者都不传，自动使用 `g_default`。

### 4.3 Global 检索

Task 执行时，若 Expert 配置了 `readGlobalActively`：
1. 查询 `global_context` 表中该 global 下的所有归档
2. 格式化为 prompt 片段注入当前 Expert
3. 受 `maxGlobalRounds` 限制检索次数

```cpp
// 替换原来 "暂无归档数据" 的占位实现
auto contexts = DataAccessFacade::getInstance()
    .getGlobalContext(ctx.globalId);
sessionHistory += formatGlobalContext(contexts);
```

**检索算法**：当前全量注入（按时间倒序）。后续可扩展为语义/向量检索。

### 4.4 Task 完成后归档

```cpp
// AgentOrchestrator 收尾时
if (!result.summary.empty()) {
    DataAccessFacade::getInstance().saveGlobalContext(
        globalId, taskId, "summary", result.summary);
}
DataAccessFacade::getInstance().saveGlobalContext(
    globalId, taskId, "plan", planJson);
if (!result.finalOutput.empty()) {
    DataAccessFacade::getInstance().saveGlobalContext(
        globalId, taskId, "output", result.finalOutput);
}
```

### 4.5 context_isolation

行为不变：`sessionHistory.clear()` 隔离前序 Expert 的 LLM 原始输出。
结构化字段（plan、goal、summary）不受影响。

## 6. 与旧版的差异

| 旧行为 | 新行为 | 原因 |
|--------|--------|------|
| `parentTaskId` 串联 Task | `global_context` 检索知识 | Task 独立，避免 LLM 原始输出污染 |
| 三态 `parent_task_id` 解析 | 直接传 `global_id` | 简化语义 |
| `loadHistoryFromParent()` 加载原始对话 | 从 `global_context` 加载结构化知识 | 语义更清晰 |
| `POST /tasks/continue` 续接 | 无续接，Task 独立 | Task 一次调用完成 |

---

## 7. 兼容性过渡

| 旧端行为 | 后端处理 | 结果 |
|----------|---------|------|
| 传 `session_id`（旧字段） | 内部映射为 `global_id` | ✅ 透明兼容 |
| 不传 `session_id` 也不传 `global_id` | 自动使用 `g_default` | ✅ 透明兼容 |
| 调 `POST /api/v1/sessions` | 内部创建 Global 记录 | ✅ 透明兼容 |
| 不传 `parent_task_id` | 通过 `global_context` 检索 | ✅ 行为升级，但对用户透明 |

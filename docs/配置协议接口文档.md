# CodePilot 配置协议接口文档

> **版本**：v1.0 | **日期**：2026-07-12
> **关联文件**：`config/experts.json`、`config/llm.json`、`config/llm.local.json`
> **核心类**：`AgentConfiguration`（`backend/agent-server/domain/agent/AgentConfiguration.h`）

---

## 一、概述

`AgentConfiguration` 是 Agent 系统配置的核心抽象。它管理 Expert 链定义、LLM 供应商配置、路由规则，并提供完整的 JSON 导入/导出接口供前端配置 UI 使用。

**核心特性**：
- **热重载**：修改 `config/experts.json` 后，每次 API 调用自动重新加载（无需重启服务）
- **单一配置入口**：所有 Expert 定义和 LLM 配置集中在 `experts.json`
- **前端就绪**：`exportJson()` / `importJson()` 接口直接对接前端配置面板

## 二、配置文件层级

```
config/
├── experts.json          ← ★ 主配置文件（Expert 链 + LLM 全局默认值）
├── llm.json              ← LLM 供应商注册表（Provider API 地址、默认模型）
├── llm.local.json        ← 本地 API Key 覆写（建议 .gitignore）
└── agent.json            ← 旧架构配置（降级兼容，不再使用）
```

### 2.1 配置优先级链

```
Expert 自身 _llm_* 字段 > experts.json 顶级 llm_* 全局默认 > llm.json 供应商默认值
```

### 2.2 前端无配置时的调试方法（组员必读）

**目前前端配置 UI 未就绪，所有配置通过直接编辑 JSON 文件完成。修改后保存即可，下一次 API 调用自动生效。**

#### 切换 LLM 供应商

编辑 `config/experts.json` 顶层字段：

```json
{
  "_llm_provider": "doubao",
  "_llm_model": "doubao-pro-32k",
  "_llm_timeout": 120,
  "_llm_temperature": 0.7,
  "experts": [...]
}
```

支持的 `_llm_provider` 值：`"doubao"`、`"deepseek"`、`"openai"`、`"qwen"`、`"zhipu"`、`"moonshot"`（对应 `config/llm.json` 中注册的 Providers）

#### 更换 API Key

编辑 `config/llm.local.json`（此文件在 `.gitignore` 中，不会被提交）：

```json
{
  "doubao": {
    "api_key": "sk-your-doubao-api-key-here"
  },
  "deepseek": {
    "api_key": "sk-your-deepseek-api-key-here"
  }
}
```

**首次使用**：复制 `config/llm.local.example.json` → `config/llm.local.json`，填入 API Key。

#### 让不同 Expert 使用不同模型

在 Expert 内部增加 `_llm_*` 字段（覆盖全局默认值）：

```json
{
  "name": "planner",
  "_llm_provider": "deepseek",
  "_llm_model": "deepseek-chat",
  "_llm_temperature": 0.3,
  ...
},
{
  "name": "executor",
  "_llm_provider": "doubao",
  "_llm_model": "doubao-pro-32k",
  "_llm_timeout": 120,
  ...
}
```

## 三、`config/experts.json` 完整字段规范

### 3.1 顶层字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:---:|--------|------|
| `_comment` | string | ❌ | — | 注释，不参与解析 |
| `_llm_provider` | string | ❌ | `""` | 全局 LLM 供应商（被 Expert 自身字段覆盖） |
| `_llm_model` | string | ❌ | `""` | 全局 LLM 模型名 |
| `_llm_timeout` | int | ❌ | `60` | 全局 LLM 超时（秒） |
| `_llm_temperature` | double | ❌ | `0.7` | 全局 LLM 温度 |
| `experts` | array | ✅ | — | Expert 定义列表 |

### 3.2 Expert 字段

详见 `docs/Expert配置说明文档.md` 第四章。此处仅列出 LLM 相关新增字段：

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:---:|--------|------|
| `_llm_provider` | string | ❌ | 继承全局 | 此 Expert 专用的 LLM 供应商 |
| `_llm_model` | string | ❌ | 继承全局 | 此 Expert 专用的 LLM 模型 |
| `_llm_timeout` | int | ❌ | 继承全局 | 此 Expert 专用的 LLM 超时 |
| `_llm_temperature` | double | ❌ | 继承全局 | 此 Expert 专用的 LLM 温度 |

## 四、AgentConfiguration API 接口（供后端 API 控制器使用）

### 4.1 类声明

```cpp
class AgentConfiguration {
public:
    static AgentConfiguration& getInstance();   // 单例

    // 初始化
    bool init(const std::string& configPath);

    // 热重载（硬编码每 call 调用一次，前端就绪后改为按需触发）
    bool reconfigure();

    // ── 前端交互接口 ──
    std::string exportJson() const;                      // → GET /api/config/experts
    bool importJson(const std::string& jsonStr);          // → PUT /api/config/experts
    std::string rawJson() const;                          // 原始 JSON（前端展示用）

    // 持久化
    bool saveToFile() const;
    bool saveToFile(const std::string& targetPath) const;

    // 查询
    const ExpertConfig* getExpert(const std::string& name) const;
    const ExpertConfig* getEntryExpert() const;
    std::vector<std::string> listExpertNames() const;
    std::vector<ExpertConfig> allExperts() const;
    bool isReady() const;
};
```

### 4.2 建议 REST API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/api/v1/config/experts` | 返回当前 Expert 配置 JSON |
| `PUT` | `/api/v1/config/experts` | 更新 Expert 配置（前端保存后调用） |
| `POST` | `/api/v1/config/experts/reload` | 强制重新加载配置文件 |

**实现示例**（待 API 控制器开发）：

```cpp
// GET /api/v1/config/experts
std::string getExpertConfig() {
    return http_response(
        AgentConfiguration::getInstance().exportJson()
    );
}

// PUT /api/v1/config/experts
std::string updateExpertConfig(const std::string& body) {
    auto& cfg = AgentConfiguration::getInstance();
    if (cfg.importJson(body) && cfg.saveToFile()) {
        return http_response(R"({"success":true})");
    }
    return http_response(
        R"({"success":false,"error":{"code":"INVALID_CONFIG","message":"配置格式无效"}})",
        "400 Bad Request"
    );
}
```

## 五、配置生效时间线

```
T0: 用户/前端 修改 config/experts.json
T1: 下一次 API 调用触发 AgentLoop::run()
T2: AgentLoop::run() 首行调用 cfg.reconfigure() ← 从磁盘重新读取
T3: LLM 调用使用最新配置
```

**注意**：当前硬编码每 call 都 reconfigure。前端就绪后，可以改为仅在 `POST /api/v1/config/experts/reload` 时触发，减少文件 I/O。

## 六、完整配置示例

### 6.1 单供应商 + 默认模型

```json
{
  "_llm_provider": "doubao",
  "_llm_model": "doubao-pro-32k",
  "_llm_timeout": 120,
  "_llm_temperature": 0.7,
  "experts": [...]
}
```

### 6.2 多供应商混合

```json
{
  "experts": [
    {
      "name": "planner",
      "_llm_provider": "deepseek",
      "_llm_model": "deepseek-chat",
      "_llm_temperature": 0.3,
      ...
    },
    {
      "name": "executor",
      "_llm_provider": "doubao",
      "_llm_model": "doubao-pro-32k",
      "_llm_timeout": 120,
      ...
    },
    {
      "name": "summarizer",
      ...
    }
  ]
}
```

### 6.3 纯降级模式（无 LLM 配置）

如果 `_llm_provider` 为空且 Expert 自身也未指定，系统降级到 `config/llm.json` 中标记为 `default: true` 的供应商。

## 总览

本次优化在**不破坏任何现有接口签名与语义**的前提下，共解决了 9 个遗留问题。所有改动均为增量添加（新增方法）或内部重写（行为不变），现有调用方无需修改任何代码即可继续使用。

---

## 优化列表

### 第 1 点 ✅ LlmClient 流式 chat

| 文件 | 改动 |
|------|------|
| `infrastructure/llm/LlmClient.h` | 新增 `OnTokenCallback` 类型别名；基类 `LlmClient` 新增 `virtual chatStream(request, onToken)` 方法，默认实现调用 `chat()` 后一次性回调全部内容 |
| `facade/LlmClientFacade.h` | 新增 `chatStream()` 公有方法声明 |
| `facade/LlmClientFacade.cpp` | 实现 `chatStream()`，自动 provider 解析 + 委托底层 |

**影响**：零破坏。`OpenAICompatibleClient` 子类可选覆盖 `chatStream()` 实现真正逐 token 推送。`MockLlmClient` 自动继承默认实现。

**使用示例**：
```cpp
LlmClientFacade::getInstance().chatStream(
    prompt,
    [](const std::string &chunk) { std::cout << chunk << std::flush; },
    "doubao"
);
```

### 第 2 点 ✅ SSE 回调抽象

| 文件 | 改动 |
|------|------|
| `facade/SSEGateway.h` | 新增 `using SendCallback` 类型别名；新增 `streamTaskEvents(SendCallback, taskId)` 新重载；旧 `streamTaskEvents(int clientFd, taskId)` 标记为 `[[deprecated]]` |
| `facade/SSEGateway.cpp` | 提取 `streamTaskEventsImpl()` 内部实现；旧接口包装为回调转发到新接口 |

**影响**：编译警告（零阻断）。调用旧接口的代码仍能编译运行，但会收到 `deprecated` 警告。HttpServer 可逐步迁移到新接口。

**使用示例**：
```cpp
// 旧接口（仍可用，但有编译警告）
gateway.streamTaskEvents(clientFd, taskId);

// 新接口（推荐）
gateway.streamTaskEvents(
    [clientFd](const std::string &frame) {
        send(clientFd, frame.data(), frame.size(), 0);
    },
    taskId
);
```

### 第 7 点 ✅ TaskController 迁移到 DataAccessFacade

| 文件 | 改动 |
|------|------|
| `api/controllers/TaskController.cpp` | 所有 `sqlite3* db` 手动管理 + `TaskService(db)` / `ToolCallRepository(db)` / `EventRepository(db)` 直接构造替换为 `DataAccessFacade::getInstance().xxx()` 单例调用 |

**具体替换**：
| 旧代码 | 新代码 |
|--------|--------|
| `openSqliteConnection(...)` + `TaskService(db).createTask(...)` + `sqlite3_close(db)` | `DataAccessFacade::getInstance().createTask(...)` |
| `ToolCallRepository(db).findByTaskId(...)` | `DataAccessFacade::getInstance().getToolCallsByTaskId(...)` |
| `EventRepository(db).findByTaskId(...)` | `DataAccessFacade::getInstance().getEventsByTaskId(...)` |
| 后台线程中 `sqlite3 *worker_db` 手动连接 | 全部通过 `DataAccessFacade::getInstance()` 单例操作 |

**影响**：后台线程不再需要独立 SQLite 连接，消除了多连接管理的复杂性和内存安全问题。

### 第 8 点 ✅ SSE progress 标准化

| 文件 | 改动 |
|------|------|
| `facade/SSEGateway.h` | 新增 `pushProgress(taskId, current, total, action, metadata)` 方法 |
| `facade/SSEGateway.cpp` | 实现 `pushProgress()`，发送 `event: progress` 事件 |

**数据格式**：
```json
event: progress
data: {
  "type": "progress",
  "content": "正在编译...",
  "metadata": {
    "channel": "status",
    "progress": { "current": 2, "total": 5, "action": "正在编译..." }
  }
}
```

**前端消费**：
```typescript
switch (event.type) {
  case "progress":
    const { current, total, action } = event.metadata.progress;
    setProgress({ current, total, action });
    break;
}
```

### 第 9 点 ✅ SSE 流式传输

| 文件 | 改动 |
|------|------|
| `facade/SSEGateway.h` | 新增 `pushDialogStream(taskId, chunk, isLast, fullContent)` 方法 |
| `facade/SSEGateway.cpp` | 实现 `pushDialogStream()`，发送 `agent_message_chunk` 事件 |

**协议**：
```
event: agent_message_chunk
data: {"type":"agent_message_chunk","content":"正在","metadata":{"channel":"dialog","streaming":true}}

event: agent_message_chunk
data: {"type":"agent_message_chunk","content":"构建方案...","metadata":{"channel":"dialog","streaming":true}}

event: agent_message                          ← isLast=true 时自动发送
data: {"type":"agent_message","content":"正在构建方案...","metadata":{"channel":"dialog","stream_end":true}}
```

**兼容性**：老前端 `EventSource` 静默忽略 `agent_message_chunk`（浏览器原生行为）。新前端额外监听 `agent_message_chunk` 事件并在 `agent_message` 到达时替换增量内容。

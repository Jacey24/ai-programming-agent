# 统一日志系统使用指南

## 1. 概述

本项目使用 **spdlog**（header-only）作为底层日志库，并通过 `modules/common/logging/` 提供了统一的封装层。

**核心原则：所有组员只需 `#include "common/logging/Logger.h"` 即可使用，无需关心 spdlog 的具体实现。**

## 2. 快速开始

```cpp
#include "common/logging/Logger.h"

void myFunction() {
    LOG_INFO("用户任务已创建，task_id={}", taskId);
    LOG_ERROR("操作失败: {}", error.what());
    LOG_DEBUG("当前参数: {} -> {}", key, value);
}
```

**输出示例（console）：**
```
[2026-07-09 10:20:30.123] [info] [default] 用户任务已创建，task_id=task_001
[2026-07-09 10:20:30.124] [error] [default] 操作失败: file not found
```

## 3. 日志级别

| 宏 | 级别 | 使用场景 |
|---|---|---|
| `LOG_TRACE(...)` | trace | 最详细的调试信息（Agent 循环每次迭代） |
| `LOG_DEBUG(...)` | debug | 调试信息（工具调用参数、上下文内容） |
| `LOG_INFO(...)` | info | 一般信息（任务状态变更、服务启动） |
| `LOG_WARN(...)` | warn | 警告信息（配置缺失但使用默认值） |
| `LOG_ERROR(...)` | err | 错误信息（操作失败、连接断开） |
| `LOG_CRITICAL(...)` | critical | 严重错误（数据库损坏、无法恢复） |

### 具名 Logger 宏（模块级独立控制）

```cpp
// 默认可用的 logger 名称：HttpServer, ToolSystem, AgentCore, 
//                            PermissionManager, Security, Database
LOGN_INFO("ToolSystem", "工具 {} 执行完成，耗时 {}ms", toolName, elapsedMs);
LOGN_ERROR("Database", "SQL 查询失败: {}", sqlError);
```

如果 `config/logging.json` 中没有某名称的 logger，会自动降级到默认 logger。

## 4. 初始化（已在 AppBootstrap 中完成，组员无需操作）

```cpp
// 在 run_agent_server() 入口处：
codepilot::log::initFromFile("./config/logging.json");
// 若文件不存在，自动使用默认配置（console + file）
```

## 5. 配置日志输出（config/logging.json）

### 默认配置

```json
{
  "loggers": [
    {
      "name": "default",
      "level": "debug",
      "sinks": ["console", "file"],
      "file_path": "/app/logs/codepilot.log",
      "max_file_size_mb": 10,
      "max_files": 5
    },
    {
      "name": "HttpServer",
      "level": "info",
      "sinks": ["console"]
    },
    {
      "name": "ToolSystem",
      "level": "debug",
      "sinks": ["console", "file"]
    },
    {
      "name": "AgentCore",
      "level": "trace",
      "sinks": ["console", "file"]
    }
  ]
}
```

### 配置字段说明

| 字段 | 说明 | 可选值 |
|---|---|---|
| `name` | Logger 名称 | 任意字符串 |
| `level` | 日志级别 | `trace`,`debug`,`info`,`warn`,`error`,`critical`,`off` |
| `sinks` | 输出目标数组 | `console`,`file`,`sqlite` |
| `pattern` | 输出格式 | spdlog pattern（见下文） |
| `file_path` | 文件路径（仅 file sink） | 绝对路径或相对路径 |
| `max_file_size_mb` | 单个日志文件最大 MB（仅 file sink） | 整数 |
| `max_files` | 保留的历史文件数（仅 file sink） | 整数 |

### 输出格式（pattern）说明

```
默认: [%Y-%m-%d %H:%M:%S.%e] [%^%l%$] [%n] %v
输出: [2026-07-09 10:20:30.123] [info] [default] 消息内容
```

- `%Y` 年，`%m` 月，`%d` 日
- `%H:%M:%S.%e` 时分秒毫秒
- `%^...%$` 根据级别着色
- `%l` 级别名称（小写）
- `%n` logger 名称
- `%v` 实际消息内容

## 6. 各场景推荐配置

### 开发调试（Docker 本地运行）

```
"AgentCore":   level=trace, sinks=[console, file]
"ToolSystem":  level=debug, sinks=[console, file]
```

### 生产环境（Docker 部署）

```
"default":     level=info,  sinks=[file]
"HttpServer":  level=warn,  sinks=[file]
```

### 单独调试某模块

临时修改 `config/logging.json` 中对应 logger 的 level 即可，无需重新编译。

## 7. 运行时动态调整

```cpp
// 仅调整某个模块的级别（运行时生效）
codepilot::log::setLevel("AgentCore", codepilot::log::Level::Debug);

// 全局调整所有 logger
codepilot::log::setLevel(codepilot::log::Level::Info);
```

## 8. 文件日志位置

- **Docker 环境**：`/app/logs/codepilot.log`（自动轮转，最多保留 5 个 10MB 文件）
- **本地 Windows**：可在 `config/logging.json` 中修改 `file_path`

## 9. 常见问题

### Q: 我不想用日志系统怎么办？
用不用都可以，不影响编译。建议逐步替换 `std::cout`，但不强制。

### Q: 如何关闭所有日志？
在 `config/logging.json` 中设置 `"level": "off"`。

### Q: 我想看更详细/更少的日志？
修改对应 logger 的 level：`trace` 最详细，`error` 最简略，`off` 关闭。

### Q: spdlog 下载失败怎么办？
`third_party/spdlog/include/` 已包含完整的 header-only 库，无需联网下载。如果仓库中没有，手动从 https://github.com/gabime/spdlog/releases 下载 v1.14.1 的 include 目录放入即可。

### Q: 文件日志报错路径不存在？
Docker 环境下 `/app/logs/` 已自动创建。Windows 环境下可以改为 `./logs/codepilot.log`。

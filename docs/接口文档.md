# AI 编程智能体接口文档

## 1. 文档说明

本文档描述 AI 编程智能体系统的后端接口设计。系统采用前后端分离架构：

* 前端：TUI Client，负责用户输入、对话展示、任务状态展示、工具输出展示、权限确认。
* 后端：Agent Server，负责会话管理、任务调度、Agent 执行、大模型调用、工具调用、权限控制、日志记录、历史回放。
* 通信方式：

  * 普通操作：HTTP REST API
  * 实时执行过程：SSE 事件流
  * 数据格式：JSON
* 数据库：SQLite
* 部署方式：Docker

---

# 2. 基础约定

## 2.1 Base URL

```text
http://127.0.0.1:8080
```

## 2.2 API 前缀

```text
/api/v1
```

## 2.3 Content-Type

所有普通 HTTP 请求与响应默认使用：

```http
Content-Type: application/json
```

SSE 事件流使用：

```http
Content-Type: text/event-stream
```

## 2.4 通用响应格式

成功响应：

```json
{
  "success": true,
  "data": {}
}
```

失败响应：

```json
{
  "success": false,
  "error": {
    "code": "TASK_NOT_FOUND",
    "message": "任务不存在",
    "details": {}
  }
}
```

## 2.5 时间格式

系统统一使用 ISO 8601 字符串：

```json
"2026-07-06T10:30:00Z"
```

也可以在 SQLite 中用整数时间戳存储，但接口层统一返回 ISO 8601。

# 3. 核心资源模型

## 3.1 Session 会话

```json
{
  "id": "session_001",
  "title": "修复 C++ 项目",
  "created_at": "2026-07-06T10:00:00Z",
  "updated_at": "2026-07-06T10:30:00Z"
}
```

## 3.2 Message 消息

```json
{
  "id": "msg_001",
  "session_id": "session_001",
  "role": "user",
  "content": "帮我修复这个项目的编译错误",
  "created_at": "2026-07-06T10:01:00Z"
}
```

`role` 可选值：

```text
system
user
assistant
tool
```

## 3.3 Task 任务

```json
{
  "id": "task_001",
  "session_id": "session_001",
  "workspace_id": "workspace_001",
  "goal": "修复 C++ 项目的编译错误",
  "status": "running",
  "created_at": "2026-07-06T10:02:00Z",
  "updated_at": "2026-07-06T10:05:00Z"
}
```

任务状态 `status` 可选值：

```text
created
planning
running
waiting_permission
completed
failed
cancelled
```

## 3.4 Workspace 工作区

```json
{
  "id": "workspace_001",
  "name": "demo-cpp-project",
  "path": "/workspace/demo-cpp-project",
  "created_at": "2026-07-06T10:00:00Z"
}
```

说明：

* Agent 只能访问 workspace 内部文件。
* 默认禁止访问 workspace 外部路径。
* 默认忽略 `.git`、`build`、`dist`、`node_modules`、二进制文件等。

## 3.5 AgentEvent 执行事件

```json
{
  "id": "event_001",
  "task_id": "task_001",
  "type": "tool_started",
  "content": "开始执行构建命令",
  "metadata": {
    "tool_name": "shell.run",
    "command": "cmake --build build"
  },
  "created_at": "2026-07-06T10:03:00Z"
}
```

事件类型 `type` 可选值：

```text
task_created
task_planning
agent_message
tool_started
tool_output
tool_finished
permission_required
permission_resolved
file_changed
task_completed
task_failed
task_cancelled
```

## 3.6 ToolCall 工具调用

```json
{
  "id": "tool_call_001",
  "task_id": "task_001",
  "tool_name": "shell.run",
  "arguments": {
    "command": "cmake --build build",
    "cwd": "/workspace/demo-cpp-project",
    "timeout": 120
  },
  "success": false,
  "result": {
    "exit_code": 1,
    "stdout": "",
    "stderr": "undefined reference to Config::load"
  },
  "created_at": "2026-07-06T10:03:00Z"
}
```

## 3.7 PermissionRequest 权限请求

```json
{
  "id": "perm_001",
  "task_id": "task_001",
  "tool_name": "shell.run",
  "risk_level": "dangerous",
  "action": "rm -rf build/",
  "reason": "清理旧构建目录",
  "status": "pending",
  "created_at": "2026-07-06T10:04:00Z"
}
```

权限状态 `status` 可选值：

```text
pending
approved
rejected
expired
```

风险等级 `risk_level` 可选值：

```text
safe
medium
dangerous
blocked
```

## 3.8 FileChange 文件变更

```json
{
  "id": "file_change_001",
  "task_id": "task_001",
  "file_path": "src/app/Config.cpp",
  "change_type": "modified",
  "diff": "--- a/src/app/Config.cpp\n+++ b/src/app/Config.cpp\n...",
  "created_at": "2026-07-06T10:06:00Z"
}
```

`change_type` 可选值：

```text
created
modified
deleted
renamed
```


查询参数：

| 参数    | 类型      | 必填 | 说明         |
| ----- | ------- | -- | ---------- |

| depth | integer | 否  | 扫描深度，默认 2  |

示例：

```http

```

响应：

```json
{
  "success": true,
  "data": {
    "path": "src",
    "items": [
      {
        "name": "main.cpp",
        "path": "src/main.cpp",
        "type": "file",
        "size": 1024
      },
      {
        "name": "app",
        "path": "src/app",
        "type": "directory"
      }
    ]
  }
}
```

## 7.6 读取工作区文件

```http
GET /api/v1/workspaces/{workspace_id}/files/content
```

查询参数：

| 参数         | 类型      | 必填 | 说明     |
| ---------- | ------- | -- | ------ |
| path       | string  | 是  | 文件相对路径 |
| start_line | integer | 否  | 起始行    |
| end_line   | integer | 否  | 结束行    |

示例：

```http
GET /api/v1/workspaces/workspace_001/files/content?path=src/main.cpp&start_line=1&end_line=80
```

响应：

```json
{
  "success": true,
  "data": {
    "path": "src/main.cpp",
    "start_line": 1,
    "end_line": 80,
    "content": "#include <iostream>\nint main() {\n  return 0;\n}\n"
  }
}
```

说明：

* 该接口只允许读取 workspace 内文件。
* 如果文件过大，后端应返回 `FILE_TOO_LARGE`。
* 如果是二进制文件，后端应返回 `BINARY_FILE_NOT_SUPPORTED`。

# 8. 任务接口

## 8.1 创建任务

```http
POST /api/v1/tasks
```

请求体：

```json
{
  "session_id": "session_001",
  "workspace_id": "workspace_001",
  "input": "帮我修复这个 C++ 项目的编译错误",
  "options": {
    "auto_run_safe_commands": true,
    "require_permission_for_file_write": true,
    "max_steps": 20
  }
}
```

字段说明：

| 字段           | 类型     | 必填 | 说明     |
| ------------ | ------ | -- | ------ |
| session_id   | string | 是  | 会话 ID  |
| workspace_id | string | 是  | 工作区 ID |
| input        | string | 是  | 用户任务输入 |
| options      | object | 否  | 执行选项   |

响应：

```json
{
  "success": true,
  "data": {
    "id": "task_001",
    "session_id": "session_001",
    "workspace_id": "workspace_001",
    "goal": "帮我修复这个 C++ 项目的编译错误",
    "status": "created",
    "created_at": "2026-07-06T10:02:00Z",
    "updated_at": "2026-07-06T10:02:00Z"
  }
}
```

说明：

* 创建任务后，后端应异步启动 Agent 执行。
* 前端应立即订阅任务事件流：`GET /api/v1/tasks/{task_id}/events`。

## 8.2 获取任务列表

```http
GET /api/v1/tasks
```

查询参数：

| 参数         | 类型      | 必填 | 说明      |
| ---------- | ------- | -- | ------- |
| session_id | string  | 否  | 按会话过滤   |
| status     | string  | 否  | 按任务状态过滤 |
| page       | integer | 否  | 页码      |
| page_size  | integer | 否  | 每页数量    |

响应：

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "id": "task_001",
        "session_id": "session_001",
        "workspace_id": "workspace_001",
        "goal": "帮我修复这个 C++ 项目的编译错误",
        "status": "completed",
        "created_at": "2026-07-06T10:02:00Z",
        "updated_at": "2026-07-06T10:08:00Z"
      }
    ],
    "page": 1,
    "page_size": 20,
    "total": 1
  }
}
```

## 8.3 获取任务详情

```http
GET /api/v1/tasks/{task_id}
```

响应：

```json
{
  "success": true,
  "data": {
    "id": "task_001",
    "session_id": "session_001",
    "workspace_id": "workspace_001",
    "goal": "帮我修复这个 C++ 项目的编译错误",
    "status": "running",
    "plan": [
      "检查项目结构",
      "运行构建命令",
      "分析错误日志",
      "定位相关代码",
      "生成并应用补丁",
      "重新构建验证"
    ],
    "current_step": "分析错误日志",
    "created_at": "2026-07-06T10:02:00Z",
    "updated_at": "2026-07-06T10:04:00Z"
  }
}
```

## 8.4 取消任务

```http
POST /api/v1/tasks/{task_id}/cancel
```

请求体：

```json
{
  "reason": "用户手动取消"
}
```

响应：

```json
{
  "success": true,
  "data": {
    "id": "task_001",
    "status": "cancelled",
    "cancelled_at": "2026-07-06T10:05:00Z"
  }
}
```

说明：

* 如果任务正在执行 Shell 命令，后端应尝试终止子进程。
* 如果任务已经 completed、failed、cancelled，再次取消应返回 `TASK_ALREADY_FINISHED`。

## 8.5 重试任务

```http
POST /api/v1/tasks/{task_id}/retry
```

请求体：

```json
{
  "from_step": 3
}
```

响应：

```json
{
  "success": true,
  "data": {
    "id": "task_002",
    "source_task_id": "task_001",
    "status": "created",
    "created_at": "2026-07-06T10:20:00Z"
  }
}
```

说明：

* 重试任务会创建一个新任务。
* 新任务可以复用原任务上下文、日志和文件分析结果。

# 9. SSE 事件流接口

## 9.1 订阅任务事件流

```http
GET /api/v1/tasks/{task_id}/events
```

响应头：

```http
Content-Type: text/event-stream
Cache-Control: no-cache
Connection: keep-alive
```

SSE 事件格式：

```text
event: tool_started
data: {"id":"event_001","task_id":"task_001","type":"tool_started","content":"开始执行构建命令","metadata":{"tool_name":"shell.run"},"created_at":"2026-07-06T10:03:00Z"}

```

## 9.2 事件示例

### task_created

```text
event: task_created
data: {"id":"event_001","task_id":"task_001","type":"task_created","content":"任务已创建","metadata":{},"created_at":"2026-07-06T10:02:00Z"}

```

### task_planning

```text
event: task_planning
data: {"id":"event_002","task_id":"task_001","type":"task_planning","content":"Agent 正在生成执行计划","metadata":{"plan":["检查项目结构","运行构建命令","分析错误日志"]},"created_at":"2026-07-06T10:02:05Z"}

```

### agent_message

```text
event: agent_message
data: {"id":"event_003","task_id":"task_001","type":"agent_message","content":"我将先运行构建命令，查看当前编译错误。","metadata":{},"created_at":"2026-07-06T10:02:10Z"}

```

### tool_started

```text
event: tool_started
data: {"id":"event_004","task_id":"task_001","type":"tool_started","content":"开始执行 Shell 命令","metadata":{"tool_name":"shell.run","command":"cmake --build build"},"created_at":"2026-07-06T10:02:20Z"}

```

### tool_output

```text
event: tool_output
data: {"id":"event_005","task_id":"task_001","type":"tool_output","content":"undefined reference to Config::load","metadata":{"stream":"stderr"},"created_at":"2026-07-06T10:02:25Z"}

```

### tool_finished

```text
event: tool_finished
data: {"id":"event_006","task_id":"task_001","type":"tool_finished","content":"Shell 命令执行完成","metadata":{"tool_name":"shell.run","exit_code":1,"duration_ms":3400},"created_at":"2026-07-06T10:02:28Z"}

```

### permission_required

```text
event: permission_required
data: {"id":"event_007","task_id":"task_001","type":"permission_required","content":"需要用户确认写入文件","metadata":{"permission_id":"perm_001","tool_name":"file.apply_patch","risk_level":"medium"},"created_at":"2026-07-06T10:03:00Z"}

```

### file_changed

```text
event: file_changed
data: {"id":"event_008","task_id":"task_001","type":"file_changed","content":"文件已修改：src/app/Config.cpp","metadata":{"file_path":"src/app/Config.cpp","change_type":"modified"},"created_at":"2026-07-06T10:04:00Z"}

```

### task_completed

```text
event: task_completed
data: {"id":"event_009","task_id":"task_001","type":"task_completed","content":"任务完成，编译错误已修复。","metadata":{"success":true},"created_at":"2026-07-06T10:08:00Z"}

```

### task_failed

```text
event: task_failed
data: {"id":"event_010","task_id":"task_001","type":"task_failed","content":"任务失败，大模型请求失败。","metadata":{"error_code":"LLM_REQUEST_FAILED"},"created_at":"2026-07-06T10:08:00Z"}

```

## 9.3 断线重连

客户端可通过 `Last-Event-ID` 请求头继续订阅：

```http
Last-Event-ID: event_005
```

后端应从该事件之后继续推送。

# 19. SQLite 数据库设计

## 19.1 sessions

```sql
CREATE TABLE sessions (
    id TEXT PRIMARY KEY,
    title TEXT NOT NULL,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL
);
```

## 19.2 messages

```sql
CREATE TABLE messages (
    id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    role TEXT NOT NULL,
    content TEXT NOT NULL,
    created_at TEXT NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id)
);
```

## 19.3 workspaces

```sql
CREATE TABLE workspaces (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    path TEXT NOT NULL,
    created_at TEXT NOT NULL
);
```

## 19.4 tasks

```sql
CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    session_id TEXT NOT NULL,
    workspace_id TEXT NOT NULL,
    goal TEXT NOT NULL,
    status TEXT NOT NULL,
    plan TEXT,
    current_step TEXT,
    created_at TEXT NOT NULL,
    updated_at TEXT NOT NULL,
    FOREIGN KEY (session_id) REFERENCES sessions(id),
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id)
);
```

说明：

* `plan` 可以用 JSON 字符串保存。

## 19.5 task_events

```sql
CREATE TABLE task_events (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    type TEXT NOT NULL,
    content TEXT NOT NULL,
    metadata TEXT,
    created_at TEXT NOT NULL,
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```

## 19.6 tool_calls

```sql
CREATE TABLE tool_calls (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    tool_name TEXT NOT NULL,
    arguments TEXT NOT NULL,
    success INTEGER NOT NULL,
    result TEXT,
    created_at TEXT NOT NULL,
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```

## 19.7 permission_requests

```sql
CREATE TABLE permission_requests (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    tool_name TEXT NOT NULL,
    risk_level TEXT NOT NULL,
    action TEXT NOT NULL,
    reason TEXT,
    status TEXT NOT NULL,
    metadata TEXT,
    created_at TEXT NOT NULL,
    resolved_at TEXT,
    FOREIGN KEY (task_id) REFERENCES tasks(id)
);
```

## 19.8 file_changes

```sql
CREATE TABLE file_changes (
    id TEXT PRIMARY KEY,
    task_id TEXT NOT NULL,
    file_path TEXT NOT NULL,
    change_type TEXT NOT NULL,
    diff TEXT,
    created_at TEXT NOT NULL,
    FOREIGN KEY (task_id) REFERENCES tasks(id)
```

## 19.9 execution_logs

# 23. 安全规则

系统必须遵守以下安全规则：

1. Agent 默认只能访问 workspace 内文件。
2. 禁止通过 `../`、软链接等方式越权访问 workspace 外部路径。
3. 写文件、删除文件、应用补丁必须经过权限模块。
4. 危险 Shell 命令必须经过用户确认。
5. 以下命令默认视为危险命令：

   * `rm -rf`
   * `sudo`
   * `chmod -R 777`
   * `chown -R`
   * `dd`
   * `mkfs`
   * `shutdown`
   * `reboot`
   * `curl | sh`
   * `wget | bash`
   * `git reset --hard`
   * `git clean -fd`
   * `git push`
6. Docker 部署时，workspace 应通过 volume 挂载到 `/workspace`。
7. SQLite 数据库应挂载到 `/data/agent.db`，避免容器删除后数据丢失。
8. LLM API Key 必须通过环境变量注入，不应写死在配置文件中。

# 25. 总结

本系统的接口设计围绕以下核心流程展开：

```text
TUI 创建会话
  ↓
TUI 创建工作区
  ↓
TUI 创建任务
  ↓
后端 Agent Server 执行任务
  ↓
后端通过 SSE 推送执行过程
  ↓
TUI 展示工具输出、任务状态和权限确认
  ↓
用户确认危险操作
  ↓
Agent 继续执行
  ↓
任务完成
  ↓
日志、工具调用、文件变更全部写入 SQLite
  ↓
用户可查看历史回放
```

最终目标是实现一个支持本地开发场景的 AI 编程智能体，具备任务理解、上下文管理、工具调用、文件修改、命令执行、权限控制、实时展示和历史回放能力。

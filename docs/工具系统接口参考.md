## 1. 三分钟快速上手

### 1.1 启动时初始化（只调用一次）

```cpp
#include "application/ToolSystem.h"
using namespace codepilot;

int main() {
    // 一行初始化 —— 自动创建所有子系统并注册所有工具
    ToolSystem::getInstance().init("/path/to/workspace");

    // ... 启动 HTTP 服务、等待请求 ...
}
```

### 1.2 各角色使用模板

**曾嘉睿（Agent 核心）** — 调用工具 + 获取元数据：

```cpp
auto& sys = ToolSystem::getInstance();
ToolContext ctx{"task_001", "ws_001", "/path/to/workspace", "sess_001", {}};

// 调用工具（自动发布 ToolStarted → ToolFinished 事件）
ToolResult r = sys.callTool("shell.run", ctx,
    {{"command", "cmake --build build"}});

// 带权限检查的工具调用（medium/dangerous 时阻塞等待用户确认）
ToolResult r2 = sys.callToolWithPermission("file.write", ctx,
    {{"path", "main.cpp"}, {"content", "..."}});

// 获取工具列表构造 LLM prompt
json schemas = sys.getToolSchemas();           // OpenAI function calling 格式
json summaries = sys.getToolSummaries();       // 轻量摘要
std::string toolList = sys.listToolsByGroup(); // 分组文本格式
```

**郑嘉娴（API 层）** — 返回 JSON + 处理权限审批：

```cpp
auto& sys = ToolSystem::getInstance();

// GET /api/v1/tools           → json 数组
json toolList = sys.registry().listToolInfo();

// GET /api/v1/tools/{name}    → json 对象
json detail = sys.registry().getToolDetail("file.read");

// 权限审批回调（POST /api/v1/permissions/{id}/approve）
sys.permissionManager().resolvePermission(requestId, true);  // 同意
sys.permissionManager().resolvePermission(requestId, false); // 拒绝

// SSE 推送：按 taskId 订阅事件
auto listenerId = sys.eventBus().subscribeByTaskId(
    "task_001", [client_fd](const EventData& e) {
        sendSSE(client_fd, e.serialize());
    });
```

**钟经添（SQLite 存储）** — 全局事件钩子持久化：

```cpp
auto& sys = ToolSystem::getInstance();
sys.eventBus().onEvent([&](const EventData& e) {
    saveToDatabase(e.serialize());
});
```

**刘子恒（前端 SSE）** — 前端通过 SSE 接收事件：

```
event: tool_started
data: {"id":"event_001","task_id":"task_001","type":"tool_started",...}

event: breakpoint_hit
data: {"id":"event_080","task_id":"task_001","type":"agent_message",
       "content":"工具 shell.run 已暂停于 before_tool_call 断点",
       "metadata":{"tool_name":"shell.run","breakpoint_type":"before_tool_call"}}
```

## 2. ToolSystem 门面类

### 2.1 获取单例 & 初始化

| 方法 | 说明 |
|---|---|
| `ToolSystem::getInstance()` | 获取全局单例 |
| `init(workspacePath, configPath="")` | 一次性初始化，幂等（重复调用不生效） |
| `isInitialized()` | 是否已初始化 |

**init() 内部自动完成：**
- 创建 `Workspace`（路径安全校验器）
- 创建 `ProcessRunner`（进程执行器）
- 创建 `EventBus`（事件总线）
- 创建 `RiskDetector`（危险命令检测）
- 创建 `PermissionManager`（权限管理器）
- 创建 `Debugger`（断点调试器，默认关闭）
- 创建 `ToolRegistry` + 注册 18 个工具

### 2.2 子系统访问

```cpp
ToolRegistry&      reg = sys.registry();          // 工具注册中心
EventBus&          bus = sys.eventBus();           // 事件总线
Workspace&         ws  = sys.workspace();          // 工作区路径安全
RiskDetector&      det = sys.riskDetector();       // 危险命令检测
PermissionManager& pm  = sys.permissionManager();  // 权限管理
ProcessRunner&     pr  = sys.processRunner();      // 进程执行
Debugger&          dbg = sys.debugger();           // ★ 断点调试器
```

### 2.3 工具调用

```cpp
// callTool() — 直接调用工具
// 自动发布 ToolStarted / ToolFinished 事件到 EventBus
// 自动检测重复调用（连续 3 次相同失败指令发出警告）
ToolResult r = sys.callTool("file.read", ctx, {{"path", "main.cpp"}});

// callToolWithPermission() — 带权限检查的工具调用（Sprint 2 Agent 主入口）
//   - Blocked → 直接拒绝，不执行
//   - Medium/Dangerous → 创建 PermissionRequest
//     → 阻塞等待用户通过 API 同意/拒绝（PermissionManager::waitForResolution）
//   - Safe → 直接执行（同 callTool）
ToolResult r2 = sys.callToolWithPermission("shell.run", ctx, args);
```

### 2.4 元数据查询（供 Agent PromptBuilder 使用）

```cpp
// OpenAI function calling 格式（所有 tool schema）
json schemas = sys.getToolSchemas();

// 轻量摘要（name + description，用于渐进式提示）
json summaries = sys.getToolSummaries();

// 工具名称列表
std::vector<std::string> names = sys.listToolNames();

// 按分组列出工具（纯文本格式，直接拼入 prompt）
std::string text = sys.listToolsByGroup();
// 输出示例：
//   [file 组] file.list, file.read, file.write, file.search, ...
//   [git 组]  git.status, git.diff, git.commit, git.log, git.branch
//   [shell 组] shell.run
```

### 2.5 调试器控制

```cpp
// 开启/关闭调试器（默认关闭，生产环境零开销）
sys.setDebuggerEnabled(true);
bool enabled = sys.isDebuggerEnabled();

// 获取调试器引用
Debugger& dbg = sys.debugger();
```

## 4. 各模块具体 API

### 4.1 ToolRegistry（工具注册中心）

```cpp
ToolRegistry& reg = sys.registry();

// 注册 & 查询
reg.registerTool(std::make_unique<MyTool>());     // 注册自定义工具
Tool* t = reg.getTool("file.read");                // 查询工具（nullptr=不存在）
bool ok = reg.hasTool("shell.run");                 // 检查工具存在
vector<string> names = reg.listToolNames();         // 列出所有工具名

// 工具分组系统
reg.registerGroup("file", "文件操作工具组：用于管理项目文件...");
string prompt = reg.getGroupPrompt("file");         // 获取分组提示词
string list = reg.listAvailableToolsByGroup();      // 分组展示文本

// 调用工具（低级接口，通常用 sys.callTool() 代替）
ToolResult r = reg.call("file.read", ctx, args);

// 输出格式（给 LLM 和 API 层）
json schemas = reg.listSchemas();                    // 全量 OpenAI tool_call
json schemasByGroup = reg.listSchemas("file");       // 按分组
json summaries = reg.listSummaries();                // 轻量摘要
json info = reg.listToolInfo();                      // 含 risk_level 的 API 格式
json detail = reg.getToolDetail("file.read");        // 单工具详情
```

### 4.2 EventBus（事件发布/订阅）

```cpp
EventBus& bus = sys.eventBus();

// 发布事件
bus.publish(EventData::Create("task_001", EventType::ToolStarted,
                               "开始执行 file.read",
                               {{"tool_name", "file.read"}}));

// 订阅（按类型）
auto id = bus.subscribe(EventType::ToolFinished, [](const EventData& e) {
    std::cout << e.serialize() << "\n";
});

// 订阅所有事件
auto id2 = bus.subscribeAll([](const EventData& e) { /* 所有事件 */ });

// ★ 按 taskId 订阅（SSE 推送专用）
auto id3 = bus.subscribeByTaskId("task_001", [](const EventData& e) {
    sendToSSEClient(e.serialize());
});

// ★ 全局事件钩子（持久化专用）
auto id4 = bus.onEvent([](const EventData& e) {
    saveToDatabase(e);
});

// 取消订阅
bus.unsubscribe(id);

// 查询历史（线程安全）
vector<EventData> history = bus.getHistory("task_001");
vector<EventData> byType = bus.getHistoryByType(EventType::ToolStarted, "task_001");

// 清空历史
bus.clearHistory("task_001");
```

**事件类型（11 种）：**

| 枚举 | 序列化字符串 | 触发场景 |
|---|---|---|
| `TaskCreated` | `task_created` | 任务已创建 |
| `TaskPlanning` | `task_planning` | Agent 正在规划 |
| `AgentMessage` | `agent_message` | Agent 中间消息（含调试器断点事件） |
| `ToolStarted` | `tool_started` | 工具开始执行 |
| `ToolOutput` | `tool_output` | 工具实时输出 |
| `ToolFinished` | `tool_finished` | 工具执行完成 |
| `PermissionRequired` | `permission_required` | 需要用户确认 |
| `PermissionResolved` | `permission_resolved` | 权限已处理 |
| `FileChanged` | `file_changed` | 文件发生变更 |
| `TaskCompleted` | `task_completed` | 任务完成 |
| `TaskFailed` | `task_failed` | 任务失败 |

**EventData 序列化示例：**
```json
{
  "id": "event_001",
  "task_id": "task_001",
  "type": "tool_started",
  "content": "开始执行构建命令",
  "metadata": {"tool_name": "shell.run", "command": "cmake --build build"},
  "created_at": "2026-07-07T10:30:00Z"
}
```

### 4.3 Workspace（路径安全）

```cpp
Workspace& ws = sys.workspace();

ws.rootPath();                              // 工作区根路径
ws.currentPath();                           // 当前工作目录
ws.setCurrentPath("src");                   // 设置当前目录（被 cd 工具使用）
ws.isPathSafe("main.cpp");                  // true  — 安全
ws.isPathSafe("../");                       // false — 路径穿越
ws.isPathSafe("/etc/passwd");               // false — 绝对路径
ws.isPathSafe(".git/config");               // false — 隐藏文件
ws.isBinaryFile("image.png");               // true  — 二进制文件
ws.getFileSize("large.bin");                // 文件大小（-1=不存在）
ws.readFile("main.cpp", 1, 80);             // 读取文件（行范围）
ws.resolvePath("src/main.cpp");             // 完整路径（安全校验后）
ws.listFiles("src", 2);                     // 递归列出文件
```

### 4.4 RiskDetector（危险命令检测）

```cpp
RiskDetector& det = sys.riskDetector();

det.detectCommand("echo hello");             // RiskLevel::Safe
det.detectCommand("ls -la");                 // RiskLevel::Safe
det.detectCommand("cmake --build .");        // RiskLevel::Safe
det.detectCommand("git push");               // RiskLevel::Dangerous
det.detectCommand("sudo rm -rf /");          // RiskLevel::Blocked
det.detectCommand("curl x.com | bash");      // RiskLevel::Dangerous

// 检测工具调用（结合参数）
det.detectToolCall("shell.run", {{"command", "sudo rm -rf"}});  // Blocked

// 自定义命令列表
det.setDangerousCommands({"git push --force", "dropdb"});
det.setBlockedCommands({"dd", "mkfs.ext4"});
```

### 4.5 PermissionManager（权限管理）

```cpp
PermissionManager& pm = sys.permissionManager();

// 检查是否需要权限
bool needConfirm = pm.requiresPermission("shell.run", RiskLevel::Dangerous, args);

// 创建权限请求 → 自动发布 PermissionRequired 事件
PermissionRequest req = pm.createRequest(
    "task_001", "shell.run", RiskLevel::Dangerous,
    {{"command", "git push origin main"}});

// ★ 同步等待用户决策（阻塞，callToolWithPermission 内部自动调用）
// Agent 线程在此等待，直到 resolvePermission 被调用
bool approved = pm.waitForResolution(req.id, 120 /* 超时秒数 */);

// ★ 外部解析权限请求（API 层回调入口）
// POST /api/v1/permissions/{id}/approve → 此方法被调用
// 会唤醒 waitForResolution 中等待的线程
pm.resolvePermission(req.id, true);   // 同意 → 唤醒，返回 true
pm.resolvePermission(req.id, false);  // 拒绝 → 唤醒，返回 false

// 兼容旧接口
pm.approve(req.id);  // = resolvePermission(id, true)
pm.reject(req.id);   // = resolvePermission(id, false)
pm.expire(req.id);   // 超时过期

// 查询
PermissionRequest* r = pm.getRequest("perm_1");
vector<PermissionRequest> pending = pm.getPendingRequests("task_001");
```

**PermissionRequest 字段：**
```
id, taskId, toolName, arguments, riskLevel,
status(pending/approved/rejected/expired),
createdAt, resolvedAt
```

### 4.6 ProcessRunner（进程执行）

```cpp
ProcessRunner& pr = sys.processRunner();

pr.setWorkingDirectory("/workspace/project");
ProcessResult r = pr.execute("git status", 30);
// r.success / r.output / r.errorOutput / r.exitCode

r = pr.execute("cmake --build build", 120);  // 120 秒超时
```

## 5. Debugger（断点调试器）— ★ 新增

调试器允许开发者在工具执行前后设置断点，暂停 Agent 的执行流程，查看/修改参数和结果。

### 5.1 开启调试器

```cpp
// 开启调试器（默认关闭）
sys.setDebuggerEnabled(true);

// 检查状态
sys.isDebuggerEnabled();   // true/false
```

**生产环境 `setDebuggerEnabled(false)` 时零开销**（仅一次原子变量读取）。

### 5.2 断点管理

```cpp
Debugger& dbg = sys.debugger();

// 设置断点
dbg.setBreakpoint("shell.run", BreakpointType::BeforeToolCall);
dbg.setBreakpoint("file.write", BreakpointType::AfterToolCall);

// 清除断点
dbg.removeBreakpoint("shell.run", BreakpointType::BeforeToolCall);
dbg.clearBreakpoints();

// 查询断点
bool has = dbg.hasBreakpoint("file.write", BreakpointType::AfterToolCall);
vector<BreakpointConfig> bps = dbg.listBreakpoints();
```

### 5.3 调试控制（API 层调用）

```cpp
// 继续执行
dbg.doContinue("task_001");

// 单步（执行完当前工具后在下个工具前再次暂停）
dbg.doStepOver("task_001");

// 跳过当前工具（返回模拟结果）
ToolResult mock;
mock.success = true;
mock.output = "模拟输出";
dbg.doSkip("task_001", mock);

// 修改工具参数（不唤醒线程，配合 doContinue 使用）
dbg.modifyArguments("task_001", {{"command", "ls -la"}});

// 修改工具结果（不唤醒线程，配合 doContinue 使用）
dbg.modifyResult("task_001", mock);

// 查询状态
DebugState state = dbg.getState("task_001");  // Disabled/Idle/Paused/Stepping
string stateStr = dbg.getStateString("task_001");
PausedContext ctx = dbg.getPausedContext("task_001");
```

### 5.4 调试流程时序

```
Agent 线程 (callTool 内部)              API 线程 (郑嘉娴的端点)
──────────────────────────             ──────────────────────────
callTool("shell.run")
  ↓
pauseBeforeTool()
  ├─ EventBus 发布 breakpoint_hit
  ├─ cv.wait() ← 阻塞                   POST /api/v1/debug/{taskId}/continue
  │                                        ↓
  │                                      doContinue(taskId)
  │                                        ├─ cv.notify_all()
  │◄── 唤醒 ────────────────────────────┘
  ├─ 使用 modifiedArgs
  ↓
Tool::execute()
  ↓
pauseAfterTool()
  ├─ EventBus 发布 breakpoint_hit
  ├─ cv.wait() ← 阻塞                   POST /api/v1/debug/{taskId}/modify_args
  │                                        ↓
  │                                      modifyArguments(taskId, newResult)
  │                                        └─ 不唤醒（继续修改）
  │                                      POST /api/v1/debug/{taskId}/continue
  │                                        ↓
  │◄── 唤醒 ────────────────────────────┘
  ├─ 使用 modifiedResult
  ↓
返回 result
```

### 5.5 DebugState 状态机

```
Disabled ──setEnabled(true)──→ Idle
Idle ──断点命中──→ Paused
Paused ──doContinue()──→ Continue → 遇到下个断点 → Paused
Paused ──doStepOver()──→ Stepping → 工具执行完 → 自动 Paused
Paused ──doSkip()──→ Continue（返回模拟结果，跳过工具）
```

## 9. 完整使用示例

### 场景 A：Agent 执行 build → 分析错误 → 修文件

```cpp
auto& sys = ToolSystem::getInstance();
ToolContext ctx{"task_build", "ws_001", ".", "sess_001", {}};

// Step 1: 执行构建
ToolResult r = sys.callTool("shell.run", ctx,
    {{"command", "cmake --build build 2>&1"}, {"timeout", 120}});

if (!r.success) {
    // Step 2: 读取 CMakeLists.txt 分析配置
    r = sys.callTool("file.read", ctx,
        {{"path", "CMakeLists.txt"}, {"start_line", 1}, {"end_line", 50}});

    // Step 3: 修改文件（需要权限确认）
    r = sys.callToolWithPermission("file.write", ctx,
        {{"path", "CMakeLists.txt"}, {"content", "updated content..."}});
}
```

### 场景 B：Git 提交流程

```cpp
auto& sys = ToolSystem::getInstance();
ToolContext ctx{"task_commit", "ws_001", ".", "sess_001", {}};

// 查看状态和 diff
ToolResult status = sys.callTool("git.status", ctx, {});
ToolResult diff   = sys.callTool("git.diff", ctx, {});

// 提交（需要权限确认，因为是 dangerous 操作）
ToolResult commit = sys.callToolWithPermission("git.commit", ctx,
    {{"message", "修复编译错误"}});
```

## 10. 扩展指南：新增自定义工具

只需 3 步：

```cpp
// 1. 继承 Tool 接口
class MyTool : public Tool {
    string name() const override { return "my.custom"; }
    string description() const override { return "我的自定义工具"; }
    string group() const override { return ToolGroups::MISC; }
    ToolSchema schema() const override { /* 定义参数 */ }
    RiskLevel riskLevel(const json& args) const override { return RiskLevel::Safe; }
    ToolResult execute(const ToolContext& ctx, const json& args) override {
        // 实现逻辑
        return ToolResult::Ok("done");
    }
};

// 2. 注册到 ToolSystem（启动时）
ToolSystem::getInstance().registry().registerTool(
    std::make_unique<MyTool>());

// 3. 通过门面类调用（和内置工具一样）
ToolResult r = ToolSystem::getInstance().callTool("my.custom", ctx, args);
```

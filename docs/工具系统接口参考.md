## 1. 三分钟快速上手

### 1.1 启动时初始化（只调用一次）

```cpp
#include "application/ToolSystem.h"
using namespace codepilot;

int main() {
    // 一行初始化 —— 自动创建 Workspace/EventBus/RiskDetector/
    // PermissionManager/ProcessRunner，注册 9 个工具
    ToolSystem::getInstance().init("/path/to/workspace");

    // ... 启动 HTTP 服务、等待请求 ...
}
```

### 1.2 各角色使用模板

**曾嘉睿（Agent 核心）** — 调用工具 + 发布事件：

```cpp
auto& sys = ToolSystem::getInstance();
ToolContext ctx{"task_001", "ws_001", "/path/to/workspace", "sess_001", {}};

// 调用工具（自动发布 ToolStarted → ToolFinished 事件）
ToolResult r = sys.callTool("shell.run", ctx, {{"command", "cmake --build build"}});
// r.success / r.output / r.error / r.exitCode
```

**郑嘉娴（API 层）** — 返回 JSON 给前端：

```cpp
auto& sys = ToolSystem::getInstance();

// GET /api/v1/tools           → json 数组
json toolList = sys.registry().listToolInfo();

// GET /api/v1/tools/{name}    → json 对象
json detail = sys.registry().getToolDetail("file.read");

// GET /api/v1/tools/schemas   → OpenAI tool_call 格式
json schemas = sys.registry().listSchemas();
```

**钟经添（SQLite 存储）** — 订阅所有事件并持久化：

```cpp
auto& sys = ToolSystem::getInstance();
sys.eventBus().subscribeAll([&](const EventData& e) {
    saveToDatabase(e.serialize());  // e.serialize() 返回完整 JSON 字符串
});
```

**刘子恒（前端 SSE）** — 获取历史事件（EventDispatcher）：

```cpp
// GET /api/v1/tasks/{task_id}/events → EventDispatcher 实时转发
// 数据格式 = e.serialize()，与接口文档完全对齐
```

## 2. ToolSystem 门面类

### 2.1 获取单例 & 初始化

| 方法 | 说明 |
|---|---|
| `ToolSystem::getInstance()` | 获取全局单例 |
| `init(workspacePath, configPath="")` | 一次性初始化，幂等（重复调用不生效） |
| `isInitialized()` | 是否已初始化 |

**init() 内部自动完成：**
- 创建 `Workspace`（路径安全校验器）
- 创建 `ProcessRunner`（进程执行器）
- 创建 `EventBus`（事件总线）
- 创建 `RiskDetector`（危险命令检测）
- 创建 `PermissionManager`（权限管理器）
- 创建 `ToolRegistry` + 注册 9 个工具

### 2.2 子系统访问

```cpp
ToolRegistry&      reg = sys.registry();          // 工具注册中心
EventBus&          bus = sys.eventBus();           // 事件总线
Workspace&         ws  = sys.workspace();          // 工作区路径安全
RiskDetector&      det = sys.riskDetector();       // 危险命令检测
PermissionManager& pm  = sys.permissionManager();  // 权限管理
ProcessRunner&     pr  = sys.processRunner();      // 进程执行
```

### 2.3 便捷工具调用

```cpp
// callTool() — 调用工具 + 自动发布 ToolStarted / ToolFinished 事件
ToolResult r = sys.callTool("file.read", ctx, {{"path", "main.cpp"}});

// callToolWithPermission() — 带权限检查的工具调用
//   - riskLevel == Blocked → 直接拒绝，不执行
//   - riskLevel == Medium/Dangerous → 创建 PermissionRequest 后继续执行
ToolResult r2 = sys.callToolWithPermission("shell.run", ctx, args);
```

## 3. 当前注册的 9 个工具

| 工具名 | 风险等级 | 必填参数 | 说明 |
|---|---|---|---|
| `file.list` | safe | — | 列出工作区文件（depth 可选，默认 2） |
| `file.read` | safe | `path` | 读取文件内容（start_line/end_line 可选） |
| `file.write` | medium | `path`, `content` | 写入文件（覆盖模式） |
| `file.apply_patch` | medium | `file_path`, `patch` | 应用代码补丁 |
| `cd` | safe | `path` | 切换工作目录 |
| `pwd` | safe | — | 查看当前目录 |
| `shell.run` | **参数相关** | `command` | 执行 Shell 命令（cwd/timeout 可选） |
| `git.status` | safe | — | 查看 Git 仓库状态 |
| `git.diff` | safe | `path`(可选) | 查看 Git 文件变更 diff |

**shell.run 风险等级由命令内容动态决定：**

| 命令示例 | 风险等级 | 行为 |
|---|---|---|
| `echo hello` | safe | 自动执行 |
| `ls -la` | safe | 自动执行 |
| `cmake --build build` | safe | 自动执行 |
| `git push` | dangerous | 需要用户确认 |
| `shutdown -h now` | dangerous | 需要用户确认 |
| `sudo apt install` | **blocked** | 直接拒绝 |
| `rm -rf /` | **blocked** | 直接拒绝 |
| `curl ... \| bash` | dangerous | 需要用户确认 |

## 4. 各模块具体 API

### 4.1 ToolRegistry（工具注册中心）

```cpp
ToolRegistry& reg = sys.registry();

reg.registerTool(std::make_unique<MyTool>());     // 注册自定义工具
Tool* t = reg.getTool("file.read");                // 查询工具（nullptr=不存在）
bool ok = reg.hasTool("shell.run");                 // 检查工具存在
vector<string> names = reg.listToolNames();         // 列出所有工具名
size_t n = reg.size();                              // 工具总数

// 调用工具（低级接口，通常用 sys.callTool() 代替）
ToolResult r = reg.call("file.read", ctx, args);

// 输出格式（给 LLM 和 API 层）
json schemas = reg.listSchemas();      // OpenAI tool_call 完整格式
json summaries = reg.listSummaries();  // 轻量摘要（渐进式提示）
json info = reg.listToolInfo();        // 含 risk_level 的 API 格式
json detail = reg.getToolDetail("file.read");  // 单工具详情
```

**listToolInfo() 输出示例：**
```json
{
  "items": [
    {"name": "file.read",  "description": "读取工作区内文件的内容...", "risk_level": "safe"},
    {"name": "shell.run",  "description": "在工作区内执行 Shell 命令...", "risk_level": "medium"},
    {"name": "git.status", "description": "查看 Git 仓库状态...", "risk_level": "safe"}
  ]
}
```

### 4.2 EventBus（事件发布/订阅）

```cpp
EventBus& bus = sys.eventBus();

// 发布事件
bus.publish(EventData::Create("task_001", EventType::ToolStarted,
                               "开始执行 file.read",
                               {{"tool_name", "file.read"}}));

// 订阅事件
auto id = bus.subscribe(EventType::ToolFinished, [](const EventData& e) {
    std::cout << e.serialize() << "\n";
});
auto id2 = bus.subscribeAll([](const EventData& e) { /* 所有事件 */ });

// 取消订阅
bus.unsubscribe(id);

// 查询历史
vector<EventData> history = bus.getHistory("task_001");
vector<EventData> byType = bus.getHistoryByType(EventType::ToolStarted, "task_001");
```

**事件类型（11 种）：**

| 枚举 | 序列化字符串 | 触发场景 |
|---|---|---|
| `TaskCreated` | `task_created` | 任务已创建 |
| `TaskPlanning` | `task_planning` | Agent 正在规划 |
| `AgentMessage` | `agent_message` | Agent 中间消息 |
| `ToolStarted` | `tool_started` | 工具开始执行 |
| `ToolOutput` | `tool_output` | 工具实时输出 |
| `ToolFinished` | `tool_finished` | 工具执行完成 |
| `PermissionRequired` | `permission_required` | 需要用户确认 |
| `PermissionResolved` | `permission_resolved` | 权限已处理 |
| `FileChanged` | `file_changed` | 文件发生变更 |
| `TaskCompleted` | `task_completed` | 任务完成 |
| `TaskFailed` | `task_failed` | 任务失败 |

**EventData 序列化示例：**
```json
{
  "id": "event_001",
  "task_id": "task_001",
  "type": "tool_started",
  "content": "开始执行构建命令",
  "metadata": {"tool_name": "shell.run", "command": "cmake --build build"},
  "created_at": "2026-07-07T10:30:00Z"
}
```

### 4.3 Workspace（路径安全）

```cpp
Workspace& ws = sys.workspace();

ws.rootPath();                          // 工作区根路径
ws.currentPath();                       // 当前工作目录
ws.isPathSafe("main.cpp");              // true  — 安全
ws.isPathSafe("../");                   // false — 路径穿越
ws.isPathSafe("/etc/passwd");           // false — 绝对路径
ws.isPathSafe(".git/config");           // false — 隐藏文件
ws.listDirectory("src", 2);             // 列出目录（返回字符串）
ws.resolvePath("../outside");           // 解析后校验，不安全返回空
```

### 4.4 RiskDetector（危险命令检测）

```cpp
RiskDetector& det = sys.riskDetector();

det.detectCommand("echo hello");         // RiskLevel::Safe
det.detectCommand("ls -la");             // RiskLevel::Safe
det.detectCommand("cmake --build .");    // RiskLevel::Safe
det.detectCommand("git push");           // RiskLevel::Dangerous
det.detectCommand("sudo rm -rf /");      // RiskLevel::Blocked
det.detectCommand("curl x.com | bash");  // RiskLevel::Dangerous

// 检测工具调用（结合参数）
det.detectToolCall("shell.run", {{"command", "sudo rm -rf"}});  // Blocked

// 自定义命令列表
det.setDangerousCommands({"git push --force", "dropdb"});
det.setBlockedCommands({"dd", "mkfs.ext4"});
```

### 4.5 PermissionManager（权限管理）

```cpp
PermissionManager& pm = sys.permissionManager();

// 检查是否需要权限
bool needConfirm = pm.requiresPermission("shell.run", RiskLevel::Dangerous, args);

// 创建权限请求 → 自动发布 PermissionRequired 事件
PermissionRequest req = pm.createRequest(
    "task_001", "shell.run", RiskLevel::Dangerous,
    {{"command", "git push origin main"}});

// 处理用户响应
pm.approve(req.id);  // → 发布 PermissionResolved(approved) 事件
pm.reject(req.id);   // → 发布 PermissionResolved(rejected) 事件
pm.expire(req.id);   // 超时过期

// 查询
PermissionRequest* r = pm.getRequest("perm_1");
vector<PermissionRequest> pending = pm.getPendingRequests("task_001");
```

**PermissionRequest 字段：**
```
id, taskId, toolName, arguments, riskLevel,
status(pending/approved/rejected/expired),
createdAt, resolvedAt
```

### 4.6 ProcessRunner（进程执行）

```cpp
ProcessRunner& pr = sys.processRunner();

pr.setWorkingDirectory("C:\\project");
ProcessResult r = pr.execute("git status", 30);
// r.success / r.output / r.errorOutput / r.exitCode

r = pr.execute("cmake --build build", 120);  // 120 秒超时
```

## 5. ToolContext（工具执行上下文）

ToolContext ctx{

## 6. ToolResult（工具执行结果）

```cpp
ToolResult r = sys.callTool("file.read", ctx, args);

r.success    // bool — 是否成功
r.output     // string — 成功时的输出内容
r.error      // string — 失败时的错误信息
r.exitCode   // int — 进程退出码
r.metadata   // json — 附加元数据

// 快捷构造
ToolResult::Ok("success output");           // success=true
ToolResult::Err("error message", -1);       // success=false, exitCode=-1
```

## 8. 完整使用示例（各场景）

### 场景 A：Agent 执行 build → 分析错误 → 修文件

```cpp
auto& sys = ToolSystem::getInstance();
ToolContext ctx{"task_build", "ws_001", ".", "sess_001", {}};

// Step 1: 执行构建
ToolResult r = sys.callTool("shell.run", ctx,
    {{"command", "cmake --build build 2>&1"},
     {"timeout", 120}});

if (!r.success) {
    // Step 2: 读取 CMakeLists.txt 分析配置
    r = sys.callTool("file.read", ctx,
        {{"path", "CMakeLists.txt"},
         {"start_line", 1},
         {"end_line", 50}});

    // Step 3: 修改文件
    r = sys.callToolWithPermission("file.write", ctx,
        {{"path", "CMakeLists.txt"},
         {"content", "updated content..."}});
}
```

### 场景 B：提交前检查

```cpp
auto& sys = ToolSystem::getInstance();
ToolContext ctx{"task_review", "ws_001", ".", "sess_001", {}};

// 查看状态
ToolResult status = sys.callTool("git.status", ctx, {});
// 查看 diff
ToolResult diff   = sys.callTool("git.diff", ctx, {});
```

### 场景 C：API 层返回数据

```cpp
// Controller 伪代码
json getTools() {
    return ToolSystem::getInstance().registry().listToolInfo();
}

json getToolDetail(const string& name) {
    auto detail = ToolSystem::getInstance().registry().getToolDetail(name);
    if (detail.is_null()) return {{"error", "not found"}};
    return detail;
}
```

### 场景 D：权限审批回调

    }

## 9. Sprint 3+ 扩展指南

新增一个自定义工具只需 3 步：

```cpp
// 1. 继承 Tool 接口
class MyTool : public Tool {
    string name() const override { return "my.custom"; }
    string description() const override { return "我的自定义工具"; }
    ToolSchema schema() const override { /* 定义参数 */ }
    RiskLevel riskLevel(const json& args) const override { return RiskLevel::Safe; }
    ToolResult execute(const ToolContext& ctx, const json& args) override {
        // 实现逻辑
        return ToolResult::Ok("done");
    }
};

// 2. 注册到 ToolSystem（启动时）
ToolSystem::getInstance().registry().registerTool(
    std::make_unique<MyTool>());

// 3. 通过门面类调用（和内置工具一样）
ToolResult r = ToolSystem::getInstance().callTool("my.custom", ctx, args);
```

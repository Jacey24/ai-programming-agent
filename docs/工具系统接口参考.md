## 1. 快速开始（30 秒）

```cpp
#include "domain/tools/Tool.h"
#include "domain/tools/ToolRegistry.h"
#include "domain/tools/BuiltinShell.h"
#include "domain/tools/FileTool.h"
#include "infrastructure/filesystem/Workspace.h"

using namespace codepilot;

// 1. 初始化
auto workspace = std::make_shared<Workspace>("/path/to/workspace");
auto shell = std::make_shared<BuiltinShell>(workspace);
ToolRegistry registry;
registerFileTools(registry, shell);

// 2. 调用工具
ToolContext ctx{"task_001", "ws_001", "/path/to/workspace", "session_001", {}};
json args;
args["path"] = "src/main.cpp";
args["start_line"] = 1;
args["end_line"] = 50;

ToolResult result = registry.call("file.read", ctx, args);
// result.success / result.output / result.error
```

## 2. 核心数据结构

### 2.1 ToolResult（工具执行结果）

```cpp
struct ToolResult {
    bool success;          // 是否成功
    std::string output;    // 成功输出
    std::string error;     // 错误信息
    json metadata;         // 元数据（任意 JSON）
    int exitCode;          // 退出码

    static ToolResult Ok(const std::string& output);
    static ToolResult Err(const std::string& error, int exitCode = -1);
};
```

**JSON 序列化示例**：
```json
{
    "success": false,
    "output": "",
    "error": "File not found: src/missing.cpp",
    "metadata": {},
    "exitCode": -4
}
```

### 2.2 ToolContext（工具执行上下文）

```cpp
struct ToolContext {
    std::string taskId;
    std::string workspaceId;
    std::string workspacePath;
    std::string sessionId;
    std::map<std::string, std::string> options;
};
```

### 2.3 EventData（事件数据结构）

对齐**接口文档.md 第3.5节**：

```cpp
struct EventData {
    std::string id;
    std::string taskId;
    EventType type;         // 枚举，详见 2.4
    std::string content;
    json metadata;          // 任意 JSON，如 {"exit_code": 1}
    std::string createdAt;  // ISO 8601 格式

    std::string typeToString() const;
    std::string serialize() const;  // 返回 JSON 字符串

    static EventData Create(const std::string& taskId, EventType type,
                            const std::string& content,
                            const json& metadata = json::object());
};
```

**JSON 序列化示例**（与接口文档完全对齐）：
```json
{
    "id": "event_001",
    "task_id": "task_001",
    "type": "tool_started",
    "content": "开始执行构建命令",
    "metadata": {"tool_name": "shell.run", "command": "cmake --build build"},
    "created_at": "2026-07-06T10:03:00Z"
}
```

### 2.4 EventType（事件类型枚举）

| 枚举值 | 序列化字符串 | 说明 |
|---|---|---|
| `TaskCreated` | `task_created` | 任务已创建 |
| `TaskPlanning` | `task_planning` | 正在规划 |
| `AgentMessage` | `agent_message` | Agent 消息 |
| `ToolStarted` | `tool_started` | 工具开始执行 |
| `ToolOutput` | `tool_output` | 工具实时输出 |
| `ToolFinished` | `tool_finished` | 工具执行完成 |
| `PermissionRequired` | `permission_required` | 需要用户确认 |
| `PermissionResolved` | `permission_resolved` | 权限已处理 |
| `FileChanged` | `file_changed` | 文件变更 |
| `TaskCompleted` | `task_completed` | 任务完成 |
| `TaskFailed` | `task_failed` | 任务失败 |
| `TaskCancelled` | `task_cancelled` | 任务取消 |

### 2.5 RiskLevel（风险等级枚举）

| 枚举值 | 序列化字符串 | 说明 |
|---|---|---|
| `Safe` | `safe` | 自动允许 |
| `Medium` | `medium` | 需要用户确认 |
| `Dangerous` | `dangerous` | 强确认 |
| `Blocked` | `blocked` | 直接拒绝 |

辅助函数：
```cpp
std::string riskLevelToString(RiskLevel level);     // "safe" / "medium" / ...
RiskLevel riskLevelFromString(const std::string& s);
```

## 3. 对外接口 — ToolRegistry

**位置**：`backend/agent-server/domain/tools/ToolRegistry.h`

这是其他模块（曾嘉睿的 Agent、郑嘉娴的 Controller）主要使用的类。

| 方法 | 返回类型 | 说明 | 使用者 |
|---|---|---|---|
| `registerTool(unique_ptr<Tool>)` | `void` | 注册工具 | 启动时调用 |
| `call(name, context, args)` | `ToolResult` | **调用工具** | **曾嘉睿(Agent)** |
| `getTool(name)` | `Tool*` | 查询工具指针 | 权限检查用 |
| `hasTool(name)` | `bool` | 检查工具存在 | 调试用 |
| `listToolNames()` | `vector<string>` | 列出工具名称 | 调试用 |
| **`listSchemas()`** | `json` | **获取所有 tool schema** | **曾嘉睿(LLM 调用)** |
| **`listSummaries()`** | `json` | **获取轻量摘要** | **曾嘉睿(渐进式提示)** |
| **`listToolInfo()`** | `json` | **工具列表（含 risk_level）** | **郑嘉娴(GET /api/v1/tools)** |
| **`getToolDetail(name)`** | `json` | **工具详情（含 schema）** | **郑嘉娴(GET /api/v1/tools/{name})** |
| `size()` | `size_t` | 注册工具数 | 调试用 |

## 4. 对外接口 — EventBus（事件发布/订阅）

**位置**：`backend/agent-server/event/EventBus.h`

**曾嘉睿（Agent）**：执行工具时调用 `bus.publish()`  
**钟经添（SQLite）**：订阅事件并持久化  
**刘子恒（前端）**：通过 SSE 接收事件

| 方法 | 说明 |
|---|---|
| `publish(event)` | 发布事件 → 自动通知所有监听器 |
| `subscribe(type, listener)` | 订阅特定类型事件，返回 listenerId |
| `subscribeAll(listener)` | 订阅所有事件 |
| `unsubscribe(id)` | 取消订阅 |
| `getHistory(taskId)` | 获取事件历史（空 = 全部） |

**Agent 使用示例**：
```cpp
// Agent 执行工具时发布事件
eventBus.publish(EventData::Create(
    taskId, EventType::ToolStarted,
    "开始执行 file.read",
    {{"tool_name", "file.read"}}
));

// 工具执行结果
ToolResult result = toolRegistry.call("file.read", ctx, args);
eventBus.publish(EventData::Create(
    taskId, EventType::ToolFinished,
    "执行完成",
    {{"tool_name", "file.read"}, {"exit_code", "0"}}
));
```

**History 查询（钟经添的 Repository）**：
```cpp
auto events = eventBus.getHistory("task_001");
for (const auto& e : events) {
    // e.serialize() 返回 JSON 字符串，可直接存入 SQLite
    saveToDatabase(e.serialize());
}
```

## 5. 对外接口 — BuiltinShell

**位置**：`backend/agent-server/domain/tools/BuiltinShell.h`

通过 `registerFileTools()` 注册到 ToolRegistry。所有 Tool 内部调用 BuiltinShell 执行实际逻辑。

| 方法 | 对应工具 | 参数 |
|---|---|---|
| `listFiles(args)` | file.list | `path`(可选), `depth`(可选, 默认2) |
| `readFile(args)` | file.read | `path`(必填), `start_line`(可选), `end_line`(可选) |
| `writeFile(args)` | file.write | `path`(必填), `content`(必填) |
| `applyPatch(args)` | file.apply_patch | `file_path`(必填), `patch`(必填) |
| `changeDirectory(args)` | cd | `path`(必填) |
| `getWorkingDirectory(args)` | pwd | 无 |
| `getSystemPrompt()` | — | 返回标准化安全规则提示词 |

## 6. 工具 Schema 定义

每个工具的 schema 在 FileTool.cpp 中用 `ToolSchema` 结构体定义：

```cpp
ToolSchema schema() const override {
    return {
        "file.read",                                     // name
        "读取工作区内文件的内容，支持指定行范围",            // description
        {
            {"path", "string", "文件相对路径", true, ""},       // {name, type, desc, required, default}
            {"start_line", "integer", "起始行号", false, "1"},
            {"end_line", "integer", "结束行号", false, "-1"}
        }
    };
}
```

### 当前已注册的 6 个工具

| 工具名 | 风险等级 | 必填参数 |
|---|---|---|
| `file.list` | safe | — |
| `file.read` | safe | path |
| `file.write` | medium | path, content |
| `file.apply_patch` | medium | file_path, patch |
| `cd` | safe | path |
| `pwd` | safe | — |

## 7. 与各模块的对接点

### 给曾嘉睿（Agent Core + LLM）

```
Agent::run() 中：
  1. 获取 tool schema → registry.listSchemas()  → 传给 LLM
  2. LLM 返回 tool_call  → 解析 tool_name + arguments
  3. 调用工具          → registry.call(name, context, args)
  4. 发布事件          → eventBus.publish(...)
  5. 获取结果          → ToolResult.success/output/error
```

### 给郑嘉娴（API 层）

```
GET /api/v1/tools          → registry.listToolInfo()
GET /api/v1/tools/{name}   → registry.getToolDetail(name)
POST /api/v1/tools/debug   → registry.call(name, context, args)
```

### 给钟经添（SQLite 存储）

```
eventBus.subscribeAll([&](const EventData& e) {
    saveToolCall(e.taskId, e.metadata["tool_name"], e.serialize());
});
```

### 给刘子恒（前端 SSE）

```
GET /api/v1/tasks/{task_id}/events  → EventDispatcher 实时转发
Event 格式 → e.serialize() 完全对齐接口文档
```

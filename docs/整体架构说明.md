# CodePilot TUI 架构说明文档

## 1. 项目概述

## 1.1 项目名称

**CodePilot TUI**

中文名：**代码领航智能体**

## 1.2 项目定位

CodePilot TUI 是一个基于 C++ 开发的 AI 编程智能体系统。系统采用前后端分离架构，前端为 TUI 终端文本交互界面，后端为独立的 Agent Server 服务。

系统能够理解用户输入的编程任务，支持多轮对话、上下文管理、任务状态跟踪、工具调用、文件读取与修改、Shell 命令执行、Web 搜索、Skill 插件调用、MCP 服务接入、危险操作权限控制、执行日志记录和历史回放。

## 1.3 核心目标

本项目的核心目标是：

```text id="7krozo"
让用户通过 TUI 输入编程任务，后端 AI Agent 能够理解任务、规划步骤、调用工具、执行本地开发操作，并将执行过程实时展示给用户。
```

系统不是普通聊天机器人，而是一个具备真实开发辅助能力的 AI Agent。

---

# 2. 架构设计目标

本项目架构设计需要满足以下目标：

## 2.1 前后端分离

前端 TUI 只负责用户交互和状态展示，后端 Agent Server 负责业务逻辑、任务调度、模型调用和工具执行。

这样可以降低模块耦合，方便后续扩展 Web 前端、VS Code 插件或远程 Agent 服务。

## 2.2 模块化设计

系统功能按照职责拆分为多个模块：

```text id="3s8ss8"
前端模块
API 接口模块
会话管理模块
任务管理模块
Agent 核心模块
上下文管理模块
工具系统模块
权限控制模块
事件推送模块
日志存储模块
插件扩展模块
部署运维模块
```

每个模块职责清晰，便于开发、测试和维护。

## 2.3 可扩展工具系统

后端通过统一的 Tool 接口注册工具，支持扩展：

```text id="cd16ho"
文件工具
Shell 工具
Git 工具
Web 搜索工具
Skill 插件工具
MCP 外部工具
```

新增工具时不需要修改 Agent 主流程，只需要实现统一接口并注册到工具中心。

## 2.4 可观察执行过程

系统需要实时展示 Agent 的执行过程，包括：

```text id="goty4d"
任务规划
工具调用
Shell 输出
文件修改
权限确认
任务完成
任务失败
```

前后端之间使用 SSE 事件流实现实时推送。

## 2.5 安全可控

由于系统可以操作本地文件和执行 Shell 命令，因此必须具备安全机制：

```text id="dp3nmo"
限制工作区访问范围
危险命令检测
文件写入确认
删除操作确认
Git 高风险操作确认
执行日志审计
```

Agent 可以建议操作，但不能绕过用户执行危险行为。

## 2.6 可持久化与可回放

系统需要将会话、任务、消息、工具调用、事件日志和文件变更保存到 SQLite 数据库中，支持历史查看和执行过程回放。

# 4. 分层架构

系统整体分为五层：

```text id="tzb0f6"

2. API Layer              接口层
3. Application Layer      应用服务层
```

## 4.1 Presentation Layer 表现层

表现层即 TUI 前端，负责用户交互。

主要职责：

```text id="qlfz4h"

展示工具执行过程
```

接口层由 Agent Server 提供 HTTP REST API 和 SSE 事件流。

主要职责：

```text id="rb76bc"

参数校验
调用应用服务
返回统一 JSON 响应
建立 SSE 事件流
处理权限确认请求
```

## 4.3 Application Layer 应用服务层

主要职责：

```text id="vyz02e"

调度 Agent 执行任务
发布任务事件
保存执行日志
```


## 4.4 Domain Layer 领域核心层

# 7. Agent 核心架构

## 7.1 Agent 核心职责

Agent Core 是系统最重要的模块，负责将用户的自然语言任务转化为可执行步骤。

核心职责：

```text id="q5nvad"
理解用户任务
生成任务计划
构建上下文
调用大模型
解析大模型返回
判断是否需要工具调用
调用工具并接收结果
根据结果继续推理
输出最终结果
```

## 7.2 Agent 执行循环

Agent 采用典型的：

```text id="gmgops"
Think → Act → Observe → Think → Act → Observe → Final
```

执行流程：

```text id="ykn0i7"
用户输入任务
  ↓
Agent 生成计划
  ↓
Agent 构造上下文
  ↓
调用 LLM
  ↓
LLM 返回下一步动作
  ↓
如果是工具调用，则执行工具
  ↓
工具结果加入上下文
  ↓
继续调用 LLM
  ↓
直到输出最终答案
```

## 7.3 Agent 主循环伪代码

```cpp id="ab7hh6"
AgentResult Agent::run(Task task) {
    context.addUserInput(task.goal);

    for (int step = 0; step < maxSteps; ++step) {
        auto prompt = promptBuilder.build(context, task);
        auto response = llm.chat(prompt, toolRegistry.schemas());

        if (response.type == "final_answer") {
            task.complete(response.content);
            return AgentResult::success(response.content);
        }

        if (response.type == "tool_call") {
            auto permission = permissionManager.check(response.toolCall);

            if (permission.needUserConfirm()) {
                eventBus.publish(PermissionRequiredEvent(permission));
                waitForUserPermission(permission.id);
            }

            auto result = toolRegistry.call(response.toolCall);
            context.addToolResult(response.toolCall, result);
            eventBus.publish(ToolFinishedEvent(result));
        }
    }

    return AgentResult::failed("超过最大执行步数");
}
```

# 8. 上下文管理架构

## 8.1 ContextManager 作用

ContextManager 负责管理发送给大模型的上下文，避免一次性把整个项目文件全部传给模型。

上下文来源包括：

```text id="v3jlpy"
系统提示词
用户当前任务
历史对话摘要
当前任务计划
已读取文件片段
工具执行结果
Shell 错误日志
文件变更 diff
项目结构摘要
```

## 8.2 上下文分类

| 类型    | 内容           | 生命周期       |
| ----- | ------------ | ---------- |
| 会话上下文 | 多轮对话消息       | 整个 Session |
| 任务上下文 | 当前任务目标、计划、步骤 | 单个 Task    |
| 文件上下文 | 读取过的文件片段和摘要  | 按需保留       |
| 工具上下文 | 工具调用结果       | 当前任务       |
| 长期摘要  | 历史任务总结       | 可持久化       |

## 8.3 上下文控制策略

为了避免上下文过大，需要采用以下策略：

```text id="54sjfj"
只读取与当前任务相关的文件
大文件只读取指定行范围
Shell 输出只保留关键错误片段
历史对话过长时生成摘要
工具结果按重要性裁剪
文件内容优先保存摘要而不是全文
```

## 8.4 Prompt 结构

推荐 Prompt 结构：

```text id="havpb4"
System Prompt

  ↓
安全规则
```

# 10. 权限控制架构

## 10.1 权限控制目标

由于 Agent 可以执行本地操作，系统必须确保所有危险行为都可控。

权限模块的目标是：

```text id="9rml4q"
允许安全操作自动执行
中风险操作提示用户确认
高风险操作强制确认
极危险操作直接阻止
```

## 10.2 权限判断流程

```text id="psgm59"
Agent 请求调用工具
  ↓
PermissionManager 检查工具和参数
  ↓
RiskDetector 判断风险等级
  ↓
如果 safe，直接执行
  ↓
如果 medium/dangerous，创建权限请求
  ↓
通过 SSE 通知前端
  ↓
前端展示确认框
  ↓
用户同意或拒绝
  ↓
后端继续执行或终止操作
```

## 10.3 风险等级

```text id="flz75x"
safe
medium
dangerous
blocked
```

| 风险等级      | 策略     |
| --------- | ------ |

# 17. 典型任务执行流程

以“修复 C++ 项目编译错误”为例：

```text id="rta3qi"
1. 用户在 TUI 输入任务
2. TUI 调用 POST /api/v1/tasks 创建任务
3. 后端创建 Task，状态为 created
4. Agent 开始规划任务，状态变为 planning
5. 后端通过 SSE 推送 task_planning 事件
6. Agent 调用 file.list 查看项目结构
7. Agent 调用 shell.run 执行构建命令
8. Shell 输出通过 tool_output 事件实时推送给前端
9. Agent 分析错误日志
10. Agent 调用 file.read 读取相关源文件
11. Agent 生成补丁
12. PermissionManager 判断写文件需要确认
13. 后端推送 permission_required 事件
14. TUI 弹出确认框
15. 用户同意
16. Agent 调用 file.apply_patch 修改文件
17. 后端记录 file_change 和 diff
18. Agent 再次调用 shell.run 验证构建
19. 构建成功
20. Agent 输出最终结果
21. 任务状态变为 completed
22. 后端保存完整日志，支持历史回放
```

# 26. 架构总结

CodePilot TUI 采用前后端分离和模块化设计。前端 TUI 负责用户交互和执行过程展示，后端 Agent Server 负责会话管理、任务调度、上下文管理、大模型调用、工具注册与执行、权限控制、日志记录和历史回放。

系统通过 HTTP REST API 完成普通请求，通过 SSE 事件流实时推送任务执行过程，通过 SQLite 持久化保存会话、任务、工具调用、权限请求、文件变更和执行日志。后端工具系统采用统一 Tool 接口，支持文件、Shell、Git、Web、Skill 和 MCP 等能力扩展。

最终系统形成一个可执行、可观察、可追踪、可扩展的 AI 编程智能体平台。
